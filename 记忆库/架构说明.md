# 长记忆模型架构说明

本文档描述本记忆库所采用的**长记忆模型**：层级划分、记忆类型、写入/检索约定及与项目其他知识源的关系。

**当前记忆默认针对 RimWorld 1.6。** 条目不特别注明时，均以 1.6 代码与 Docs 为核对依据。

---

## 一、记忆层级

| 层级 | 名称 | 说明 | 存放位置 |
|------|------|------|----------|
| **L1** | 工作记忆 | 当前会话内的临时上下文，不持久化 | 会话/对话上下文 |
| **L2** | 情景记忆 | 与具体任务/问题绑定的结论与待办 | `decisions/`、部分 `design/` |
| **L3** | 语义记忆 | 可复用的概念、约定、公式与术语 | `domain/`、`conventions/` |
| **L4** | 长期档案 | 完整规格、逻辑图、清单等权威文档 | `记忆库/docs/` |

记忆库承载 **L2、L3、L4**：L4 文档已迁入 `记忆库/docs/`，与 design/domain/decisions 同库管理。

---

## 二、记忆类型与 schema

每种记忆类型对应一种「条目」结构，详见 [schema.md](schema.md)。此处仅作类型列表：

| 类型 | 说明 | 目录 | 示例 |
|------|------|------|------|
| **设计决策** | 为什么这样实现、与文档的对应 | `design/` | 双池按 PairIndex 分组的原因 |
| **项目约定** | 命名空间、EMDefOf、Harmony、设置等 | `conventions/` | 本 mod Def 一律走 EMDefOf |
| **领域概念** | 术语定义、公式、与 记忆库/docs 的章节引用 | `domain/` | L、E_tol、双池、吸收延迟 |
| **关键决策** | ADR 风格：背景、选项、结论、后果 | `decisions/` | 采用 30 tick 进水周期 |

---

## 三、写入约定

- **一条记忆一个主题**：单文件可含多条，但每条应有清晰标题或锚点，便于 index 链接。
- **必填**：标题、类型、日期（或版本/提交范围）；**可选**：标签、关联 记忆库/docs 章节、相关文件路径。
- **不复制大段规格**：用「见 记忆库/docs/xxx 第 N 节」代替全文粘贴；记忆库写摘要与结论。
- **可追溯**：若结论来自某次讨论或代码审查，在条目中注明来源（如「逻辑审计结论」「四层模型升级待办」）。

---

## 四、检索方式

1. **按类型**：通过 [index.md](index.md) 按 `design / conventions / domain / decisions` 浏览。
2. **按标签**：在 index 或各目录 00-概览 中使用 `#标签` 聚合（如 `#双池` `#耐受` `#EMDefOf`）。
3. **按文档**：需要完整公式或清单时，从记忆库条目中的「关联文档」跳转到 `记忆库/docs/`。
4. **语义检索**：AI 或工具可对记忆库做关键词/向量检索时，优先命中标题与类型，再展开正文。

## 五、与其它知识源的关系

```
                    ┌─────────────────┐
                    │   .cursor/      │
                    │   skills/       │  ← 操作规范（如何写代码、如何检查）
                    └────────┬────────┘
                             │ 引用
                             ▼
┌──────────────────┐     ┌─────────────────┐     ┌──────────────┐
│ 记忆库/docs/     │◀────│   记忆库/       │────▶│  Source/     │
│ 规格/逻辑图/清单  │ 引用 │ 设计/约定/领域  │ 引用 │ 代码实现     │
└──────────────────┘     └─────────────────┘     └──────────────┘
```

- **记忆库** 统一承载 **docs（规格）+ design/conventions/domain/decisions（摘要与决策）**。
- **Skills** 规定「怎么做」；记忆库记录「为什么」和「是什么」（概念与约定）。
- 代码注释中可写「见 记忆库/design/xxx」或「见 记忆库/docs/泌乳系统逻辑图 第 N 节」。

---

## 六、版本与更新

- 记忆库无单独版本号；随项目一起由 Git 管理。
- 当设计或约定发生非向后兼容变更时，在对应条目或 `decisions/` 中记录原因与迁移说明。
