# 记忆库（Memory Bank）

本目录为 **rjw-cummilk** 项目的长记忆库，用于持久化设计决策、领域知识、约定与术语，便于 AI 助手与人类在跨会话中一致地复用上下文。

**当前记忆默认针对 RimWorld 1.6。** 多版本分支时，若某条仅适用于某版本，请在条目标题或文首注明。

**检索入口**：涉及泌乳、双池、耐受、设置或领域术语时，先看 [index.md](index.md) 与 [domain/00-概览.md](domain/00-概览.md)。

## 用途

- **设计决策**：为什么这样实现、与文档/规格的对应关系。
- **约定与规范**：命名空间、EMDefOf、翻译、Harmony、设置等（与 `.cursor/skills` 互补）。
- **领域概念**：泌乳系统术语（L、双池、E_tol、吸收延迟等）与公式的权威说明入口。
- **索引与检索**：通过 [index.md](index.md) 和 [schema.md](schema.md) 快速定位记忆类型与条目。

## 目录结构

```
记忆库/
├── README.md           # 本说明
├── 架构说明.md         # 长记忆模型层级与检索方式
├── schema.md           # 记忆条目的字段与类型约定
├── index.md            # 记忆索引（按类型/标签的目录、推荐标签表）
├── changelog-memory.md # 记忆库本身变更记录
├── docs/               # 完整规格文档（泌乳逻辑图、参数表、清单等，由原 Docs/ 迁移）
├── design/             # 设计决策与实现依据
├── conventions/        # 项目约定与规范摘要
├── domain/             # 领域概念与术语（含快速参考）
└── decisions/          # 关键决策记录（ADR 风格可选）
```

## 使用方式

1. **人类**：在对应子目录下新增或编辑 `.md` 文件，在 `index.md` 中补充条目链接。
2. **AI**：已通过 **`.cursor/rules/use-memory-bank.mdc`** 配置：在回答涉及设计、约定或领域知识时，优先读取 `记忆库/` 下相关文件；新增重要结论时可写入对应目录并在 index 中登记。
3. **与规格文档的关系**：完整规格、逻辑图、清单等存放在 **`记忆库/docs/`**（由原项目 `Docs/` 迁移而来）。记忆库做摘要与索引，详细规格以 `记忆库/docs/` 为准；记忆条目可引用 `docs/xxx` 并注明章节。

## 维护

- 新增记忆类型或子目录时，更新本 README 与 `架构说明.md`、`index.md`。
- 重大约定变更时，同步更新 `conventions/` 与 `.cursor/skills` 中的技能文件。
- **何时写入/更新**：实现或重构做出非显然选择时；与规格/Docs 的对应关系需要固定时；某次审查或讨论得出可复用结论时。修改与 L/双池/耐受/进水/选侧等相关的代码时，顺带检查并更新记忆库中对应条目。
- **废弃**：某条决策被推翻时，在原文顶部注明「已废弃，由 新条目（相对路径）替代」，并在 index 中该行标注「(已废弃)」。

### 命名与收口约定（整理时遵守）

| 项目 | 约定 |
|------|------|
| **命名** | `design/` 下用简短中文主题名，如 `双池与PairIndex.md`；`decisions/` 下用 `ADR-XXX-短标题.md`（如 ADR-001-进水与衰减周期）；`docs/` 保持与内容一致的中文文件名；`domain/`、`conventions/` 用主题名或 `00-概览.md`。新增条目文件名避免空格，用数字/连字符即可。 |
| **收口** | 规格类完整文档（逻辑图、参数表、清单、模拟、脚本）**仅放在 `记忆库/docs/`**，不在 design/domain 下重复放全文。关键决策（ADR）**仅在 `decisions/`**。设计结论摘要与索引在 design/；领域术语与公式权威引用在 domain/；约定摘要在 conventions/。记忆库内引用 Docs 时统一用 `记忆库/docs/xxx` 或相对路径 `docs/xxx`。 |
| **最后核对** | 重要条目（与代码强相关的 design/decisions、index、docs/README、schema）在文末可加「最后与代码核对：YYYY-MM」或「最后整理：YYYY-MM」；大改代码或记忆库后顺手更新，便于保鲜。 |

更多建议见 [建议与改进.md](建议与改进.md)。
