# 排乳反馈与原有系统不兼容说明

- **状态**：已识别
- **关联**：[激素模型-催乳素与排乳反馈](激素模型-催乳素与排乳反馈.md)、[泌乳系统逻辑图](../docs/泌乳系统逻辑图.md)、ADR-001

---

## 一、原有系统的契约

在**未**引入「排乳反馈」之前，L 衰减的契约是：

| 项目 | 内容 |
|------|------|
| **L 衰减** | 每 200 tick 执行一次，`L −= D(L,E)×step`，仅与当前 L、耐受 E、时间步长有关 |
| **泌乳期长度** | 由初始 L0 与 D(L,E) 决定，**与是否挤奶/吸奶无关** |
| **挤奶/吸奶** | 仅从双池取奶（扣池水位、可选抬 R）；**不参与** L 的衰减公式 |

即：**「泌乳能维持多久」只由进水（服药/分娩）和纯时间衰减决定，玩家排不排乳、排多排少都不改变 L 的衰减规则。**

---

## 二、后来添加的逻辑（排乳反馈）

在「激素模型」中新增：

| 新增项 | 作用 |
|--------|------|
| **lastDrainTick** | 仅在挤奶/吸奶（及可选 RJW 性行为）时由 `NotifyDrained()` 更新 |
| **days_since_drain** | (now − lastDrainTick)/60000，参与 D_eff |
| **长期不排乳抑制** | SuppressionFactor = α×days^β → D_eff 变大 → L 掉得更快 → **自然结束** |
| **StimulusBuffer** | 排乳时 Buffer+ΔB，D_eff 除以 (1+γ×Buffer) → 频繁排乳时衰减变慢 → **维持 ↑** |
| **排乳时 L+** | 可选 `AddMilkingLStimulus`，直接 L+ |

效果：**长期不排乳 → 自然结束；频繁排乳 → 维持 ↑**，并可选「雌激素/行为刺激」说明、RJW 性行为视为排乳刺激。

---

## 三、不兼容点

| 维度 | 原有系统 | 开启排乳反馈后 |
|------|----------|----------------|
| **泌乳期长度** | 仅由 L0、D、时间决定 | 还受**是否/何时排乳**影响 |
| **不挤奶** | L 按固定 D 衰减，时长可预期 | lastDrainTick 不更新 → days 大 → 抑制强 → **更快结束** |
| **频繁挤奶** | 与 L 衰减无关 | Buffer↑、lastDrainTick 常新 → 衰减慢 → **明显延长** |

因此：

- 与原有「**仅时间衰减**」的契约不一致。
- 依赖「服药/分娩后大致固定泌乳时长」的数值或玩法会被打破。
- 「雌激素/行为刺激」等文案与可选 RJW 调用 `NotifyDrained` 进一步把「排乳」与「维持」绑定，和「原有系统里排乳只取奶、不参与 L 规则」的设计冲突。

---

## 四、缓解与建议

1. **恢复原行为**：关闭「长期不排乳抑制」（`enableLactationSuppressionFromNoDrain = false`）且关闭「排乳刺激缓冲」（`enableStimulusBuffer = false`）时，D_eff = D，行为与原有系统一致。**当前默认已为两者 false，即默认恢复原逻辑。**
2. **默认与文档**：若希望默认兼容原有预期，可将上述两项默认改为 `false`，或在设置/文档中明确标注「排乳反馈」为可选扩展，与「仅时间衰减」二选一。
3. **设计文档**：在 [激素模型-催乳素与排乳反馈](激素模型-催乳素与排乳反馈.md) 中注明与原有系统的不兼容性，并引用本文。

---

## 五、小结

- **原有系统**：L 仅按时间衰减；挤奶/吸奶不参与 L 规则。
- **排乳反馈**：挤奶/吸奶触发 lastDrainTick、Buffer（及可选 L+），使「长期不排乳→自然结束、频繁排乳→维持↑」，与原有契约不兼容。
- **可选做法**：通过设置关闭抑制与缓冲以恢复原行为；或在默认/文档中明确「排乳反馈」为可选扩展并说明不兼容。
