# 奶与奶制品权限开放接口

## 用途

供其他 mod（或本 mod 内扩展）**覆盖或扩展**「谁可以使用我的奶 / 谁可以食用我的奶制品」的权限逻辑，而不必改内置名单（allowedSucklers / allowedConsumers）或默认规则。

## 位置

`MilkCum.Fluids.Lactation.Helpers.MilkPermissionExtensions`

## 两个委托列表

1. **AllowSucklerHandlers**  
   - 类型：`List<Func<Pawn producer, Pawn doer, bool?>>`  
   - 含义：产主 `producer` 是否允许 `doer` 使用自己的奶（直接吸奶或挤奶）。  
   - 返回值：`true` = 允许，`false` = 不允许，`null` = 交回内置逻辑（allowedSucklers 名单，空则默认子女+伴侣）。  
   - 调用时机：`IsAllowedSuckler(producer, doer)` 内部，**先**按顺序执行各 handler，第一个返回非 null 的结果即采用；若全部返回 null，再走内置逻辑。

2. **CanConsumeMilkProductHandlers**  
   - 类型：`List<Func<Pawn producer, Pawn consumer, Thing food, bool?>>`  
   - 含义：产主 `producer` 是否允许 `consumer` 食用自己的奶制品（带 `CompShowProducer` 的奶/精液制品）。  
   - 返回值：同上，`null` 表示交回内置逻辑（allowedConsumers 名单，空=仅产主本人）。  
   - 调用时机：扩展方法 `CanConsumeMilkProduct(consumer, food)` 内部，在已解析出 `producer` 且「非本人食用」时，**先**执行各 handler，再走内置逻辑。

## 使用示例（其他 mod）

在 mod 的静态构造或 DefsLoaded 中注册一个 handler，例如统一把「伴侣」也算作允许吸奶/食用奶制品：

```csharp
// 吸奶/挤奶：若 doer 是 producer 的伴侣，则允许（覆盖或补充名单逻辑）
MilkPermissionExtensions.AllowSucklerHandlers.Add((producer, doer) =>
{
    if (producer?.relations?.DirectRelationExists(PawnRelationDefOf.Lover, doer) == true) return true;
    if (producer?.relations?.DirectRelationExists(PawnRelationDefOf.Spouse, doer) == true) return true;
    return null; // 其他情况交回内置
});

// 奶制品食用：同上
MilkPermissionExtensions.CanConsumeMilkProductHandlers.Add((producer, consumer, food) =>
{
    if (producer?.relations?.DirectRelationExists(PawnRelationDefOf.Lover, consumer) == true) return true;
    if (producer?.relations?.DirectRelationExists(PawnRelationDefOf.Spouse, consumer) == true) return true;
    return null;
});
```

## 约定

- 多个 handler 按注册顺序调用，**第一个返回非 null 的结果即采用**，后续 handler 不再执行。  
- 单个 handler 抛异常时会被捕获，不阻断其他 handler 与内置逻辑。  
- 内置逻辑仍为：吸奶/挤奶看 allowedSucklers（空=子女+伴侣）；奶制品食用看 allowedConsumers（空=仅产主本人）。

## 默认名单排除父母

- 默认“可使用我的奶”名单**仅包含子女 + 伴侣**，**明确排除父母**（母亲/父亲）。  
- `GetDefaultSucklers` 中不会加入任何“是产主父母”的 pawn；`IsAllowedSuckler` 在名单为空时若 doer 是产主的父母则直接返回 false。  
- 扩展方法 `IsParentOf(producer, doer)` 用于判断 doer 是否为 producer 的父母（DirectRelation Parent）。  
- 中国家族称谓 mod（KinshipLabelsZH）提供 `IsAllowedToUseMilkByDefault(producer, doer)`，并在加载时通过反射注册到本 mod 的 AllowSucklerHandlers / CanConsumeMilkProductHandlers，实现“用家族关系判断：仅子女/配偶允许，父母不允许”。
