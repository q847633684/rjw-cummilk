# Equal Milking 专业级设置 UI 重构方案

按「系统类型」分层、权限独立、高频靠前、同类聚合的专业模组通用原则，将设置窗口重构为 7 大主 Tab。本文档记录原则、目标结构、与当前实现的映射及落地建议。

---

## 一、核心设计理念（专业模组通用原则）

### 原则 1：按「系统类型」分层，而不是按「功能来源」分层

| 初级设计（按来源） | 专业设计（按类型） |
|-------------------|--------------------|
| 母乳 / 精液 / 基因 / 权限 | 核心机制 / 健康风险 / 权限规则 / 数值平衡 / 模组联动 / 数据种族 / 调试工具 |

玩家更容易理解「我想改哪一类体验」。

### 原则 2：权限必须独立

权限类内容应回答：谁可以用、谁不能用、菜单是否显示、默认行为是什么。不应混入玩法机制页面。

### 原则 3：高频设置靠前，低频设置折叠

玩家最常调整：是否启用、谁可以使用、数值强度。复杂机制应放入二级层级。

### 原则 4：同类机制必须聚合

| 类型 | 必须放一起 |
|------|------------|
| 疾病 | 乳腺炎 / 卫生 / 感染 |
| 身份 | 殖民者 / 奴隶 / 囚犯 / 动物 |
| 数值 | 容量 / 流速 / 增益倍率 |

分散会严重增加理解成本。

## 二、目标主 Tab 结构（7 大系统）

```
┌────────────────────────────────────────────────────────────────────────────────────┐
│ 核心机制 │ 健康风险 │ 权限规则 │ 数值平衡 │ 模组联动 │ 数据种族 │ 调试工具 │
└────────────────────────────────────────────────────────────────────────────────────┘
```

### ① 核心机制（Core Systems）

玩家：我想控制玩法本身怎么运作。

| 子 Tab | 内容 |
|--------|------|
| 哺乳系统 | 哺乳总览 + 按类型哺乳（人形/动物/机械） |
| 精液系统 | Cumpilation 设置 |
| 流体行为 | 填充 / 覆盖 / 收集 / 外溢机制（含妹汁占位） |

特点：高频使用、玩法入口、新手最常访问。

### ② 健康风险（Health & Complications）

玩家：我想控制是否真实 & 是否有代价。

| 子 Tab | 内容 |
|--------|------|
| 乳腺炎系统 | 启用、MTB、风险倍率 |
| 卫生系统 | DBH / 房间清洁度影响 |
| 耐受系统 | 耐受降低产量、指数曲线 |
| 溢出与污染 | 污物类型、环境污染 |

特点：机制型玩家常调、沉浸党核心区域。

### ③ 权限规则（Permissions & Policies）

玩家：我想控制谁可以参与。

| 子 Tab | 内容 |
|--------|------|
| 身份权限 | 殖民者 / 奴隶 / 囚犯 / 动物 / 机械 |
| 菜单显示 | 是否显示操作菜单（催乳素/挤奶等） |
| 默认行为 | 自我挤奶 / 可被喂奶默认值 |

特点：强规则区、多人殖民地管理核心。

### ④ 数值平衡（Balance & Scaling）

玩家：我觉得太强 / 太弱。

| 子 Tab | 内容 |
|--------|------|
| 乳房与容量 | 容量系数、体积增长（RJW 乳房/对称双池） |
| 产出效率 | 流速倍率、泌乳增益 |
| 生命周期 | 基准泌乳天数、分娩增益 |
| AI 行为 | 优先挤满池、智能策略 |

特点：数值党最爱、影响整体平衡。

### ⑤ 模组联动（Integrations）

玩家：我装了其它模组，想联动。

| 子 Tab | 内容 |
|--------|------|
| RJW 联动 | 性需求、怀孕率、性行为影响 |
| DBH 联动 | 卫生系统接入 |
| 营养系统 | 食物 ↔ 乳池转换（营养→能量、回缩吸收等） |

特点：高级玩家区域、依赖外部模组。

### ⑥ 数据种族（Data & Races）

玩家：我想改规则表。

| 子 Tab | 内容 |
|--------|------|
| 产奶表 | 可产奶种族列表 |
| 奶标签 | 标签规则 |
| 种族覆盖 | 特殊种族例外 |
| 基因规则 | 基因影响泌乳 |

特点：数据驱动区、偏 MOD 制作者。

### ⑦ 调试工具（Dev & Diagnostics）

仅 DevMode 显示。子 Tab：泌乳监控、Pawn 调试、Tick 日志等。普通玩家不可见。

## 三、当前实现 → 目标结构映射

便于实施时「从哪里搬到哪里」。

| 目标主 Tab | 目标子 Tab | 当前位置（方案 B） |
|------------|------------|---------------------|
| **核心机制** | 哺乳系统 | Lactation 子 Tab 0（总览）+ 1（哺乳按类型）→ Widget_BreastfeedSettings |
| 核心机制 | 精液系统 | Cum 子 Tab 0 → Widget_CumpilationSettings |
| 核心机制 | 流体行为 | Cum 子 Tab 1（妹汁占位）；Cumpilation 中填充/覆盖/收集可考虑拆出或保留在精液 |
| **健康风险** | 乳腺炎 | Lactation 子 Tab 2 → DrawHealthSection(0) |
| 健康风险 | 卫生系统 | Lactation 子 Tab 3 → DrawHealthSection(1) DBH |
| 健康风险 | 耐受系统 | Lactation 子 Tab 4 前半 → DrawHealthSection(2) 耐受相关 |
| 健康风险 | 溢出与污染 | Lactation 子 Tab 4 后半 → 溢出污物、AI 优先满池等 |
| **权限规则** | 身份权限 + 菜单显示 + 默认行为 | PermissionsAndDefaults 整页 + Lactation 子 Tab 5（身份与菜单）→ 合并为权限规则主 Tab 下 3 子 Tab |
| **数值平衡** | 乳房与容量 / 产出效率 / 生命周期 / AI 行为 | Lactation 子 Tab 6（乳房与池）拆或保留；子 Tab 4 中 AI 行为可归入数值平衡 |
| **模组联动** | RJW / DBH / 营养系统 | Lactation 子 Tab 7（RJW）；DBH 已在健康；营养→能量/回缩在哺乳总览 → 可拆到「模组联动」下营养子 Tab |
| **数据种族** | 产奶表 / 奶标签 / 种族覆盖 / 基因规则 | GenesAndRaces 主 Tab 下 4 子 Tab → 整体迁入「数据种族」 |
| **调试工具** | 泌乳监控等 | 当前在 GenesAndRaces 子 Tab 3 下方 DrawDevModeSection → 独立主 Tab，仅 DevMode 显示 |

## 五、落地建议

| 步骤 | 内容 |
|------|------|
| **枚举与 Keyed** | 新增 `MainTabIndex` 枚举 7 项（或替换现有 4 项）；新增 `EM.Tab.CoreSystems`、`EM.Tab.HealthRisk`、`EM.Tab.Permissions`、`EM.Tab.Balance`、`EM.Tab.Integrations`、`EM.Tab.DataRaces`、`EM.Tab.DevTools`；子 Tab Keyed 按上表补全。 |
| **GetSubTabs / DoWindowContents** | 按 7 主 Tab 重写 `GetSubTabs()` 返回值；`DoWindowContents` 中 `switch (mainTabIndex)` 分支改为 7 路，内容分发到现有 Widget 或新 Widget。 |
| **内容迁移** | 健康风险：保持 DrawHealthSection 三个块，可拆为 4 子 Tab（乳腺炎/卫生/耐受/溢出）。权限规则：合并当前「权限与默认」页与 Lactation 的「身份与菜单」。数值平衡：以当前「乳房与池」为基础，拆出「产出效率」「生命周期」「AI 行为」或保留一页多区块。 |
| **调试工具** | 仅当 `Prefs.DevMode` 时在主 Tab 栏显示第 7 个 Tab；内容为现有 DrawDevModeSection 或扩展。 |
| **文档同步** | 实施后更新 [模组设置窗口完整UI结构](模组设置窗口完整UI结构.md)、[设置UI重构框架](../docs/设置UI重构框架.md)、[模组设置UI系统整理与优化方案](模组设置UI系统整理与优化方案.md)。 |

## 六、专业性体现

| 维度 | 提升点 |
|------|--------|
| 玩家体验 | 查找成本 ↓，按「类型」找设置 |
| 扩展性 | 新系统可独立加入对应主 Tab |
| 代码维护 | 主 Tab 与子 Tab 一一对应，减少深层 switch 嵌套 |
| 模块解耦 | 权限 / 数值 / 机制 / 数据分离 |
| 专业观感 | 接近大型 DLC 级模组设置结构 |
