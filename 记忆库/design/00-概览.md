# 设计决策（design）概览

本目录存放**设计决策**类记忆：为什么采用当前实现、与规格/文档的对应关系、以及曾考虑过的替代方案。

## 何时写入

- 实现或重构时做了非显然的技术选择（如双池按 PairIndex 分组、30 tick 进水周期）。
- 与《泌乳系统逻辑图》《游戏已接管变量与机制清单》等 Docs 的对应关系需要固定下来。
- 某次代码审查或逻辑审计得出结论，需要跨会话保留。

## 条目结构建议

- **title**：简短结论，如「双池按对分组的原因」。
- **context**：当时要解决的问题（如「多对乳房需要独立左右池」）。
- **conclusion**：采用的方案（如「每个 RJW 乳房 hediff 对应一对，PairIndex 一致」）。
- **rationale**：理由（性能、与 Docs 一致、可扩展等）。
- **related_docs**：链接到 `记忆库/docs/泌乳系统逻辑图.md` 等对应章节。

## 与本项目 Docs 的对应

规格文档统一放在 **`记忆库/docs/`**；以下为可沉淀设计记忆的 Doc 与示例。

| Docs 文档 | 可沉淀的设计记忆示例 |
|-----------|----------------------|
| 泌乳系统逻辑图 | L/双池/耐受/进水/衰减公式、选侧规则 |
| 游戏已接管变量与机制清单 | 哪些由原版/RJW 接管、本 mod 只读或挂接 |
| 四层模型升级-代码待办 | 升级相关决策与未采纳方案 |
| 代码建议与扩展方向 | 已采纳的扩展点与实现方式 |

（引用时用 `记忆库/docs/泌乳系统逻辑图.md` 或相对路径 `../docs/泌乳系统逻辑图.md`。）
