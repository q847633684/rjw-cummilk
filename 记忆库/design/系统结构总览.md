# 系统结构总览

本 mod 按「流体系统 → 三大体液 → 各体液下子系统」划分。下表在您给出的骨架基础上补充了各节点对应的职责与代码落点，便于按系统查改。

---

## 一、母乳系统（乳汁 / Lactation）

整体逻辑：**吃药 → 泌乳 → 乳房胀大 → 溢出 → 回缩**。下列按该因果链组织。

### 药物/分娩

药物（催乳素等）与分娩作为进水与 Lactating 的触发来源。落点：HediffWithComps_MilkCumLactating、Hediff_ProlactinTolerance、Hediff_LactatingGain、Jobs（InjectLactatingDrug）、Thoughts（成瘾/戒断/耐受/长期药物）；分娩触发见 RJW/集成。

### 泌乳系统

| 层级 | 内容补充（职责与落点） |
|------|------------------------|
| **吸收延迟** | 服药后延迟挂 Lactating、到点才进水。落点：WorldComponent_MilkCumAbsorptionDelay、HediffComp_AbsorptionDelayTip、吸收延迟 tick 计算与 Def 参数。 |
| **进水系统** | 进水来源：吃药、分娩、性刺激、挤奶和吸奶刺激。进水量受种族、营养、耐受、喷乳反射 R、炎症等影响。L（泌乳量）与 L 衰减、L→池的 TickGrowth。落点：HediffWithComps_MilkCumLactating、Hediff_ProlactinTolerance、Fluids/Shared/Helpers/FoodEnergyHelper、Harmony 营养相关 Patch、RJWSexAndFertility（性刺激）、OnGathered/OnGatheredLetdownByKeys（挤奶/吸奶刺激）。 |
| **池系统** | 多乳房多池（左/右、按对）、池容量、池上限、存储奶量；压力因子、R 对流速倍率、选侧条目。落点：CompEquallyMilkable、LactationPoolState/BreastPoolEntry（Shared）、PawnMilkPoolExtensions、CompPartialMilk、CompCumBucketLink。 |
| **出水系统** | 对池扣奶：挤奶、吸奶、回缩。挤奶：殖民者/动物取奶（DrainForConsume）、选侧规则；吸奶：哺乳（婴儿/成人）；回缩：池满或压力导致回缩，回缩吸收营养（GetReabsorbedNutritionPerDay）。落点：DrainForConsume、WorkGiver_MilkCumMilk、WorkGiver_MilkCumMilkEntity、JobDriver_MilkCumMilk、JobGiver_MilkCumBreastfeed、JobGiver_ActiveSuckle、JobDriver_ActiveSuckle、JobDriver_ForcedBreastfeed、ChildcareHelper、BreastfeedPatch、FloatMenuOptionProvider_Breastfeed、Building_Milking、RJW GatherHumanBodyResources 兼容；回缩与 GetReabsorbedNutritionPerDay 在 HediffWithComps_MilkCumLactating、CompEquallyMilkable。 |

### 思想系统（每个思想对心情的影响写在对应项下）

| 思想 | 职责与心情影响 | 落点 |
|------|----------------|------|
| **成瘾满足/戒断** | 成瘾 Hediff 两阶段（满足 +1、戒断 -12）由原版 ThoughtWorker_Hediff 驱动。 | EM_ProlactinAddictionThought（hediff=EM_Prolactin_Addiction）。 |
| **高耐受低奶** | 高耐受低奶 Thought 及 mood 效果。 | ThoughtWorker_HighToleranceLowMilk。 |
| **满池** | 满池 Thought 及 mood 效果。 | ThoughtWorker_MilkPoolFull。 |
| **长期未挤奶** | 长期未挤奶 Thought 及 mood 效果。 | ThoughtWorker_LongTimeNotMilked。 |
| **长期药物泌乳** | 长期药物泌乳 Thought 及 mood 效果。 | ThoughtWorker_LongTermDrugLactation。 |

RJW 哺乳后 Need_Sex 满足、可选 EM_HadSexWhileLactating 记忆：RJWLustIntegration.GiveSexSatisfactionAfterNursing。

### 悬停系统

健康页悬停、池满度/剩余天数/流速/产奶效率等 tip。落点：CompTipStringExtra、CompLabelInBracketsExtra、PawnColumnWorker_* 的 tip、TooltipHandler.TipRegion（设置页）、Keyed 文案（MilkFlowDrive、PoolRemainingDays 等）。

### 满池信件

池满时发送信件提醒（可开关）。落点：MilkCumSettings.enableFullPoolLetter、CompEquallyMilkable.lastFullPoolLetterTick、TrySendFullPoolLetter。

## 九、建议（如何消化第八节）

**原则**：总览以「按系统查改」为主，能并入主树的尽量并入，减少第八节长度；跨系统的集中成节更清晰。

**建议一：立刻并入主树（小改）**

| 项 | 做法 |
|----|------|
| 满池信件 | 已并入第一节「泌乳系统」下的「满池信件」行。 |
| Lactation/Data | 第一节「泌乳系统」表可增一行「**数据与 Def 扩展**」：Gene_MilkTypeData、MilkTag、RaceMilkType、Breastfeed、AsyncCollection；MilkSettings 见第六节权限。 |
| 基因/种族设置 UI | 已并入第四节「模组设置 UI」下的「基因/种族设置」行。 |
| UI 细目 | 第四节已加「其它 UI 落点见 Source/索引 或第八节」。 |

**建议二：新开节（中等改）**

| 节 | 内容 |
|----|------|
| **集成与补丁** | 新开一节（可放在「建筑和物品」之后）：(1) Integration：RJW、VCE、PipeSystem、DubsBadHygiene 及落点；(2) Harmony 非 Lactation：EventTrigger、RecipeProductProducerPatch、MilkProductConsumptionPatch、Compatibility 下各 Patch。查「和 RJW 怎么接」「兼容性补丁在哪」一眼可见。 |
| **共享流体层** | 新开一小节（如放在第二节精液之后）：Fluids/Shared 的职责（池状态/标签、流速/压力/营养模型、TextureHelper）及 Data/Models/Helpers 落点。第一节「池」「营养」可写「见共享流体层」。 |

**建议三：保留在第八节**

- **Debug/DevMode**：不并入 UI 节，避免臃肿；不单开节（太碎）。保留在第八节表格即可。
- **Core 基础设施**：不单开节；在「与目录/文档的对应」前加一句「模组入口与基础设施（ModInit、MilkCumMod、MilkCumDefOf、EventHelper 等）见 Source/索引 与第八节」。
- **精液子模块细目**：第二节保持「Cumflation、Gathering、Leaking、Bukkake」概括；在第二节表下加一句「Reactions、Oscillation、Slug、Biosculptor、Common 等细目见第八节或 Source/索引」，不把整张细目表搬进第二节。

**执行顺序**：建议一中有部分已随本次骨架调整并入；Lactation/Data 可择机补一行。建议二、三按需执行；第八节保留 Debug、Core、精液细目三类并注明「详见本节表格」。
