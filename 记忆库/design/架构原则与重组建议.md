# 架构原则与重组建议

本文档汇总**整体架构原则**与**具体文件夹/模块重组建议**，供后续重构或新功能时参考。实施时建议分阶段、小步验证，避免一次性大动导致回归。

---

## 一、整体原则

### 1. SOLID 与分离关注点（SoC）

- **现状**：已按功能分组（Core、Comps、Jobs、Helpers 等），是好的起点。
- **建议**：
  - 单文件/单类尽量**单一责任**；例如 Helpers 中的 `ExtensionHelper.cs` 混杂判断、权限、容量计算，可拆为：`PawnMilkExtensions.cs`（泌乳判断）、`MilkPermissionExtensions.cs`（权限）、`MilkCapacityHelper.cs`（容量/流速相关）等。
  - **Data** 目录只放数据模型与纯数据结构，**行为逻辑**放到 Helpers 或独立 Services；避免 Data 中的类型既持数据又写业务。

### 2. 模块化与子模块

- **Cumpilation**：已是大型子模块（Bukkake、Cumflation、Leaking 等），可考虑：
  - 独立子项目或 NuGet 包，便于复用或分离维护；
  - 或保持现状，在仓库内用清晰边界（如 CumpilationCore 入口、子文件夹按领域划分）。
- **集成代码**：RJW、PipeSystem 已收口到 **目录** `Source/Integrations/`（`Integrations/RJW`、`Integrations/PipeSystem`），命名空间未改，仅目录整理便于查找。

### 3. 命名空间标准化

- **建议**：与 mod 名称和功能层级一致，例如：
  - `RJWCumMilk.Core`（或保留 `MilkCum.Core` 以减小改动面）
  - `RJWCumMilk.Milking.Comps`、`RJWCumMilk.Milking.Models`
- **层级**：命名空间尽量**镜像文件夹结构**，便于 IDE 导航与新人理解。

### 4. 文档与索引

- **现状**：Markdown 索引（如 `Source/索引.md`、记忆库）已有，便于定位。
- **建议**：
  - 索引可考虑**半自动化生成**（如 Doxygen 或自定义脚本从 XML 注释/程序集生成摘要）。
  - 在文档中增加 **UML 或依赖关系图**（如 Core ← Milking ← UI、Patches 依赖关系）。
  - **代码注释**：统一 XML 文档注释；每个文件开头可标注主要依赖的文件/类，便于追踪。

### 5. 性能与兼容性

- **Harmony**：Patch 文件较多时，建议按**主题合并**（如 `MilkingPatches.cs`、`BreastfeedPatches.cs`），减少类型数量与潜在冲突；同时保留清晰的 `PatchManager` 或按主题的注册点，便于启用/禁用。
- **兼容性**：重构后需验证与 **RimWorld 更新**、**RJW 版本**的兼容性；关键路径（双池、挤奶、吸收延迟）建议在发布说明中注明测试环境。

### 6. 工具与自动化

- **重构**：使用 IDE 重构工具（如 Visual Studio / Rider 的“移动类型”“重命名”）或 ReSharper，减少手工改命名空间/using 的错误。
- **测试**：为核心逻辑引入**单元测试**（如 NUnit），优先覆盖：`LactationPoolState.TickGrowth`、进水/衰减公式、选侧与 Drain 逻辑；RimWorld 依赖可 Mock 或使用轻量测试装配件。

## 二、具体文件夹/模块重组建议（目标结构示例）

以下为**目标结构示例**，实施时可分阶段迁移，不必一次到位。

| 顶级文件夹 | 内容与迁移来源 | 说明 |
|------------|----------------|------|
| **Core** | 模组入口、全局设置（当前 MilkCum/Core）；Constants、VersionDiffHelper 从 Helpers 移入 | 可增加 `ModInitializer.cs` 统一 Init 与事件钩子 |
| **Milking** | 泌乳核心：合并 Comps、Data、Helpers 中泌乳相关 | 子文件夹：**Models**（BreastPoolEntry、LactationPoolState、MilkSetting）、**Components**（CompEquallyMilkable、HediffWithComps_EqualMilkingLactating）、**Utilities**（泌乳判断、FoodEnergyHelper、MilkRelatedHealthHelper 等） |
| **Breastfeeding** | 哺乳与消费：从 Helpers、Jobs、Givers 提取 Breastfeed、JobDriver_ActiveSuckle、JobGiver 等 | 可增加 `BreastfeedManager.cs` 协调权限（如 AllowedToBreastFeed） |
| **JobsAndGivers** | 所有 JobDriver、WorkGiver、JobGiver | 子文件夹建议：MilkingJobs、FeedingJobs 等，避免单目录文件过多 |
| **Patches** | 所有 Harmony 补丁（从 MilkCum/Milk/HarmonyPatches 迁出） | 子文件夹：Milking、Breastfeed、Compatibility；建议 `PatchManager.cs` 统一注册 |
| **UI** | 从 MilkCum/UI 迁移，结构可基本保持 | 可增加 `UIManager.cs` 统一窗口/列注册 |
| **ThoughtsAndEffects** | Thoughts + World 组件（ThoughtWorker、WorldComponent_EqualMilkingAbsorptionDelay） | 可增加 Effects 子文件夹便于扩展新 Hediff/效果 |
| **Cumpilation** | 保持独立模块，子文件夹已较清晰 | 可增加 `CumpilationCore.cs` 作为入口；规模大时可考虑独立仓库 |
| **Integrations** | 第三方集成（新文件夹） | 子文件夹：RJW、PipeSystem、DubsBadHygiene；每集成可设 `IntegrationInitializer` 检查 mod 存在并应用 Patch |
| **Shared** | 跨模块共享：Lang、TextureHelper、EventHelper、GeneHelper 等从 Helpers 移入 | 减少跨模块直接依赖，便于复用 |

**注意**：当前实际结构为 `MilkCum/Core`、`MilkCum/Milk/Comps` 等，上述“重组”涉及大量文件移动与命名空间变更，建议：  
- 先在小范围试点（如只拆 ExtensionHelper 或只迁 Integrations）；  
- 每步完成后跑通游戏与关键流程，再继续下一步。

## 四、推荐实施顺序（建议）

1. **低风险、高收益**：拆分 `ExtensionHelper` 中“泌乳判断 / 权限 / 容量”为若干小类（同命名空间内即可），不强制改文件夹。
2. **集成收口**：将 RJW、PipeSystem、DubsBadHygiene 归入 `Integrations` 或同一命名空间下，便于查找与开关。
3. **Patch 合并**：按主题合并 Harmony 类，并统一注册入口。
4. **命名空间与目录**：若需与“RJWCumMilk”或新层级对齐，放在上述稳定后再做，并配合 IDE 重构工具。
5. **测试与文档**：为 LactationPoolState、Drain、进水公式等增加单元测试；索引/架构图可随改动逐步补全。

上述顺序可根据人力与风险偏好调整；文档与索引的自动化、Cumpilation 独立仓库等可作为更长期目标。

## 五、实施记录

- **ExtensionHelper 拆分**（已完成）：已拆为 `PawnMilkStateExtensions`（泌乳状态、默认设置、IsLactating/IsInLactatingState 及缓存）、`MilkPermissionExtensions`（Allow*、Allowed*、名单、GetBedOwner、IsAdult、ChildcareText 等）、`PawnMilkPoolExtensions`（池条目、容量、MilkDef/MilkAmount、流速倍率、身体部位、DistanceTo 等）；`ExtensionHelper` 仅保留 `CompEquallyMilkable(ThingWithComps)` 与 `Set(CompProperties_Milkable)`。调用处已从 `ExtensionHelper.xxx` 改为对应新类（如 `MilkPermissionExtensions.GetDefaultSucklers`、`PawnMilkStateExtensions.ClearLactatingStateCache`）；RJW Harmony 的 Patch 目标已改为 `PawnMilkPoolExtensions.MilkAmount` / `MilkDef`。见 `Source/索引.md` Helpers 节与 `记忆库/docs/待办清单.md`。
- **Integrations 收口**：已将 RJW、PipeSystem **目录**移入 `Source/Integrations/`，仅改 csproj 路径，未改命名空间；索引见 `记忆库/docs/Integrations索引.md`。
- **Harmony 按主题**：仅保留文档 `记忆库/docs/HarmonyPatch主题清单.md` 供查阅；主 mod 仍为程序集扫描应用 Patch，未在代码中加 Registry。
- **目录结构梳理**：以 `Source/索引.md` 为基准对目录层级与代码结构做系统对照；在索引顶部增加「目录层级与约定」、补全 Core 遗漏的 `StatWorker_PawnOnlySafe`；新增 `记忆库/design/Source目录结构梳理.md` 固化层级、命名空间对应与后续新增文件约定。未做文件移动。
