# [design] 泌乳逻辑前提：仅在有乳房时进行

- **type**: design
- **date**: 2025-03
- **tags**: #双池 #泌乳前提 #乳房 #GetBreastList #GetBreastPoolEntries
- **related_docs**: [泌乳系统逻辑图](../docs/泌乳系统逻辑图.md)、[双池与PairIndex](双池与PairIndex.md)

## 设计前提

**逻辑是基于「胸部部位有乳房」的前提：只有在有乳房的情况下才会进行泌乳逻辑。**

- 「有乳房」= 该部位上存在乳房 hediff（如 RJW 的乳房尺寸 hediff），即 `GetBreastList()` 非空且 `GetBreastPoolEntries(pawn)` 返回非空列表。
- 无乳房 hediff 时：不创建乳池（无 Left_Default/Right_Default）、容量与流速倍率为 0、不显示乳房悬停奶量；泌乳 hediff（Lactating）可存在，但进水/挤奶/吸奶等池相关逻辑因无池条目而不产生实际效果。

## 结论

- 乳池、容量、流速、健康页乳房悬停奶量等，均以「存在 GetBreastPoolEntries 条目」为前提。
- 无 RJW 乳房 hediff 时不做「默认一对」兜底，保持空池/0 容量/0 流速。

## 代码约定与优化

- **`GetBreastPoolEntries(pawn)` 约定永不返回 null**：无乳房时返回空列表，调用方只需判断 `entries.Count == 0`，可省略 `entries == null`。
- **入口统一拦截**：`CompEquallyMilkable.Active` 增加条件 `pawn.GetBreastPoolEntries().Count > 0`，无乳房时 Active 为 false，`CompTick` 内不执行 `UpdateMilkPools`、`TryTriggerMastitis`、`UpdateHealthHediffs`，实现「无乳房不执行逻辑」。`UpdateMilkPools` 内不再做无乳池早退。
- **对外接口保留判空**：`GetReabsorbedNutritionPerDay`、`DrainForConsume`、Hediff 的 GetLetdownReflex 等由 UI/喂奶等外部调用，可能面对「有 Lactating 但无乳池」的 pawn，故仍保留 `entries.Count == 0` 的返回默认值。

## 健康页与泌乳逻辑的乳房数据一致性

**结论：健康页获取的乳房信息和泌乳逻辑里用的是同一套。**

- **唯一数据源**：`pawn.GetBreastPoolEntries()`（ExtensionHelper），内部只从 `pawn.GetBreastList()` 构建：每个乳房 hediff → `key = Part.def.defName ?? def.defName+"_"+i`，再生成 `key_L` / `key_R`、Capacity（Severity×coeff）、FlowMultiplier、PairIndex。
- **泌乳逻辑**：`UpdateMilkPools`、`DrainForConsume`、`SyncLeftRightFromBreastFullness`、`GetReabsorbedNutritionPerDay` 等都用 `Pawn.GetBreastPoolEntries()`，水位存于 `breastFullness[key]`，key 与上述一致。
- **健康页悬停身体部位**：`GetPoolEntriesForBreastPart(pawn, part)` → `GetPoolEntriesForPoolKey(pawn, part.def.defName)` → 遍历 `pawn.GetBreastPoolEntries()`，按 `e.Key == poolKey || e.Key == poolKey+"_L" || e.Key == poolKey+"_R"` 过滤，奶量用 `comp.GetFullnessForKey(e.Key)`（同一 `breastFullness`）。
- **健康页悬停乳房 hediff**：`GetPoolKeyForBreastHediff(pawn, h)` 的 key 规则与 `GetBreastPoolEntries` 内一致（`Part.def.defName ?? def.defName+"_"+IndexOf`），再走 `GetPoolEntriesForPoolKey`，同样来自 `GetBreastPoolEntries()`。

因此：键名、容量、左右、对序号、水位均来自同一套 `GetBreastPoolEntries()` 与 `breastFullness`，健康页只是按当前悬停的部位/hediff 做过滤展示。
