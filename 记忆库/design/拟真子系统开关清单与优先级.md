# 拟真子系统：开关清单与默认优先级

**目标**：在「优先拟真」前提下，把可拆分的生理隐喻落成**独立子系统**，每个子系统对应**一个总开关**（或 Dev/高级折叠内开关），便于 A/B、平衡与文档维护。  
**边界**：不追求真实乳腺的全微观仿真；以 **因果可解释、可调参、与现有双池/L/炎症线兼容** 为准。权威公式仍以 `记忆库/docs/泌乳系统逻辑图.md` 为基准，本表是**增量拟真**的规划索引。

**默认策略（实现）**：`MilkCumSettings.Realism` 中各 SYS 布尔总开关**默认全开**（字段初值 + `Scribe_Values` 缺省 `true`）；玩家/MOD 设置里**已保存**的值优先生效。旧行为需要时在高级设置中逐项关闭。

## 1. 子系统总表

| ID | 子系统 | 拟真意图 | 建议设置键前缀（示意） | 主要依赖 | 与现有实现关系 | 主要风险 |
|----|--------|----------|------------------------|----------|----------------|----------|
| **SYS-01** | **非线性顺应性进水** | 越接近容积上限，等量压力带来的体积增量越小（顺应性变差）；进水有效速率随满度下降。 | `Realism.ComplianceInflow` | 满度、可选 stretch 缓冲 | 与 `压力因子`、`GetPressureFactor` 可合并或串联；需在 `Inflow`/pool tick 一处收口 | 与旧存档「线性体感」差异大，需默认软化或兼容档 |
| **SYS-02** | **形变缓冲 + RJW 里程碑** | 超「基础容量」先进组织形变（缓冲容积），形变达基容的一定比例后**停灌**并走溢出；对 RJW **积分达阈值再改 severity**，避免每 tick 微抖。 | `Realism.StretchBuffer` `Realism.RjwStretchMilestone` | RJW 乳房 Hediff、`RJWLactatingBreastSize` 等 | 与现有「撑大 severity」「永久里程碑」部分重叠，需**统一一条整合路径** | Hediff 频繁写入、与他 mod 冲突；存档与性能 |
| **SYS-03** | **代谢门控与池内抵扣** | 显著饥饿/脱水时：**暂停或削减进水**；缺口（如本 tick 需 0.3）优先从**乳房存量**按规则抵扣营养/水（动态比例，少用固定常量）。 | `Realism.MetabolicGate` `Realism.PoolFedSelf` | `Need_Food`、`Need_Water`（及 DBH） | 与 `逻辑审阅-营养与泌乳一致性`、现有回缩/营养线衔接 | 全地图 pawn 每 tick 细算成本；需 LOD/节流 |
| **SYS-04** | **可解释高压漏奶** | 单次进水中有**基准漏率**（由满度、瞬时压、可选炎症乘子决定），再叠小随机；损失**不**纯 RNG。满度越高，期望损失越大。 | `Realism.ReflexLeak` | 满度、压力、可选 `I` | 与现有溢出污物、满池信件等联动 | 玩家觉得「奶被偷」；需 UI/一次性说明或 Dev 明细 |
| **SYS-05** | **排空—淤积反馈** | 适度排空 → 短窗内略升合成/进水效率；长期高满少排空 → 淤积权重升（接现有炎症 stasis 项，避免重复算伤）。 | `Realism.EmptyStasisCoupling` | 炎症模型、`inflammationStasis*` | 与 `HediffWithComps_MilkCumLactating` 内 stasis 挂钩 | 与乳腺炎难度强耦合，需联动表统一 |
| **SYS-06** | **刺激源分化（婴 vs 机）** | 吮吸 vs 机械挤奶对射乳/瞬时效率不同（不必拆两条池，只调 letdown/刺激项）。 | `Realism.StimulusSource` | letdown、JobDriver | 与现有 `milkingLStimulus*` 扩展 | 工作类型判定边界情况多 |
| **SYS-07** | **疼痛/心情 → 射乳抑制** | 高疼痛或极差心情抑制 letdown 或瞬时流速（粗耦合即可）。 | `Realism.StressLetdown` | `pain`、`mood` | Harmony 或 CompTick 读 need | 与战斗/特性交互需测试 |
| **SYS-08** | **断奶慢变** | 停止哺乳/药效结束后 L 非阶跃掉线，而用数日衰减曲线（可选）。 | `Realism.WeaningCurve` | L、Severity | `SeverityPerDay` 等已由游戏接管部分需对齐清单 | 与药品/永久泌乳冲突需规则 |

## 2. 推荐落地顺序（性价比 / 拟真增益）

### P0 — 先做（投入相对可控，拟真叙事提升最明显）

1. **SYS-01 非线性顺应性进水**  
   - 一条主曲线收口，不先动 RJW。玩家立刻感到「越胀越难灌」，和示意图/压力叙事一致。

2. **SYS-03 代谢门控 + 池内抵扣**  
   - 直接对应你描述的「饿/渴停灌、用已有奶补缺口」；实现上要定**优先级表**（见 §3），但与 RimWorld Need 天然贴合。

3. **SYS-04 可解释高压漏奶**  
   - 在随机层外包一层「由满度与压解释的期望」，防作弊感，并与污物/品质若有的扩展兼容。

### P1 — 第二轮（与你设想中的「撑大—停灌—溢出」强相关，但工程量大）

4. **SYS-02 形变缓冲 + RJW 里程碑**  
   - 建议拆两期：**2a** 仅 mod 内 stretch 缓冲与停灌/溢出；**2b** 再对接 RJW severity 里程碑，降低一次性风险。

5. **SYS-05 排空—淤积反馈**  
   - 在已有炎症 stasis 上收紧因果，避免「重复伤害通道」。

### P2 —  polish / 差异化玩法

6. **SYS-06 刺激源分化**  
7. **SYS-07 疼痛/心情抑制**  
8. **SYS-08 断奶曲线**  
9. **SYS-09 昼夜**（新开默认开；不需要时在拟真里关）

## 3. 进水与抵扣：建议的优先级表（草案）

> 用于 SYS-03 与 SYS-02「停灌」合并时减少歧义。数值仅为设计占位，落地时写入 `参数联动表`。

1. **硬停**：昏迷、死亡、无乳房组件等（现有前提）。  
2. **代谢停/减流**：饥饿或脱水超过阈值 → **削减或暂停进水**（可分级：先减流至 0，再只允许池内抵扣）。  
3. **容积/形变停灌**：达到「基容 + 允许形变比例」→ **禁止进水**，溢出走污物/SYS-04。  
4. **炎性/医疗暂停（可选）**：特定 Hediff 时减流（P2）。  

**池内抵扣**：在单 tick 内，对 `Need.CurLevel` 缺口换算为「可从乳池扣的最大量」，**min(缺口, 池内可动用部分)**，并扣减对应池条目；不足部分仍走原版饥饿/渴。

## 4. 与你口述链路的映射

| 你的设想 | 对应子系统 | 备注 |
|----------|------------|------|
| 基容 1.2、每秒进水 0.2 | SYS-01 + 现有 inflow | 0.2 会随满度被顺应性打折 |
| 满基容后撑大、向 RJW 传形变 | SYS-02 | 里程碑式写 severity |
| 撑到基的 20% 停灌、溢出污 | SYS-02 + 溢出 + SYS-04 | 「20%」建议做成设置项 |
| 回缩：营养/水需求补偿 | SYS-03 或独立 `Reabsorb` 扩展 | 与现有回缩逻辑合并，避免双倍 |
| 饿/渴停灌、池内动态抵扣 | SYS-03 | 按缺口比例，忌固定 0.2 |
| 进水概率漏 10–50%、满则更易且更重 | SYS-04 | 期望由满度函数给出，RNG 只加小方差 |
