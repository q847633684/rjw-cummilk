# 记忆条目 Schema

本文档定义记忆库中**记忆条目的通用结构**与各类型的必填/可选字段，便于统一写入与检索。

---

## 一、通用字段（所有类型）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| **title** | 字符串 | 是 | 记忆条目标题，用于 index 与检索 |
| **type** | 枚举 | 是 | `design` / `convention` / `domain` / `decision` |
| **date** | 日期或版本 | 建议 | 写入或最后更新日期；可为版本号或 commit 范围 |
| **tags** | 字符串数组 | 否 | 如 `#双池` `#耐受` `#EMDefOf`，用于聚合 |
| **related_docs** | 链接列表 | 否 | 关联的规格文档路径与章节，如 `记忆库/docs/泌乳系统逻辑图.md#二、端到端逻辑链` |
| **related_files** | 路径列表 | 否 | 相关源码或 Def 路径，便于跳转 |
| **最后与代码核对** | 日期或 commit | 建议 | 重要条目建议填写，便于保鲜；格式如 2025-03 或 commit 短哈希 |

正文使用 Markdown；可含表格、列表、代码块与锚点。

---

## 二、按类型的扩展字段

### 2.1 design（设计决策）

| 字段 | 必填 | 说明 |
|------|------|------|
| **context** | 建议 | 当时要解决的问题或需求 |
| **conclusion** | 是 | 采用的方案或结论 |
| **rationale** | 建议 | 理由（性能、兼容、与 Docs 一致等） |
| **alternatives** | 否 | 考虑过但未采用的方案 |

### 2.2 convention（项目约定）

| 字段 | 必填 | 说明 |
|------|------|------|
| **scope** | 建议 | 适用范围：C# / Defs / 翻译 / Harmony / 设置 |
| **rule** | 是 | 约定内容（可与 .cursor/skills 一致或摘要） |
| **examples** | 否 | 正确/错误示例或链接到技能文件 |

### 2.3 domain（领域概念）

| 字段 | 必填 | 说明 |
|------|------|------|
| **term** | 是 | 术语或符号（如 L、E_tol、双池） |
| **definition** | 是 | 简短定义或公式 |
| **authoritative_ref** | 建议 | 权威出处：`记忆库/docs/xxx` 第 N 节 |
| **related_terms** | 否 | 相关术语列表 |

### 2.4 decision（关键决策，ADR 风格）

| 字段 | 必填 | 说明 |
|------|------|------|
| **status** | 是 | 如：已采纳 / 已废弃 / 待定 |
| **context** | 是 | 背景与待解决问题 |
| **options** | 建议 | 考虑的选项简述 |
| **decision** | 是 | 最终决定 |
| **consequences** | 否 | 后果、迁移或兼容说明 |

## 三、单条记忆的 Markdown 模板示例

```markdown
## [类型] 标题

- **type**: design | convention | domain | decision
- **date**: YYYY-MM-DD 或版本
- **tags**: #标签1 #标签2
- **related_docs**: `记忆库/docs/xxx.md#锚点`

（正文：context / conclusion / definition / decision 等）

### 关联文件
- `Source/MilkCum/...`
```

实际书写时不必严格按表格填满，但**标题、类型、正文结论**应清晰可辨，以便 index 与检索。

---

## 四、与 index 的对应

- 每条在记忆库中可被索引的条目，应在 [index.md](index.md) 中有一行链接（按类型分节）。
- index 中可只写标题 + 相对路径，必要时加 1 行摘要或 tags。

---

## 五、废弃与替代约定

- 某条决策或设计被推翻时，**在原文顶部**注明：`**已废弃**，由新条目标题（相对路径）替代。`
- 在 [index.md](index.md) 中该行链接后标注「(已废弃)」，便于检索时区分。
- 不删除旧条目，保留以便追溯「为何不再采用」。
