# 吃药 n 次、不同时间 —— 药效剩余时间与泌乳量 L 模拟

脚本 `drug_n_doses_simulate.py` 按**当前代码与 Def** 模拟：在**不同时间**吃 **n 次**药后，**药效剩余时间**（≈ L/日衰减）与 **泌乳量 L** 随时间的变化。

---

## 公式与约定（与代码一致）

| 项 | 说明 |
|----|------|
| 吸收延迟 | 服药后 **0.25 游戏日** 才计入 L（BaseAbsorptionDelayTicks = 15000 tick） |
| 进水 | 每剂 **ΔL = raw × E_tol(t_before) × C_dose**，C_dose=1；Prolactin raw=0.5，Lucilactin raw=2 |
| 耐受 | E_tol(t) = max(1−t, 0.05)；每剂 Prolactin +0.044、Lucilactin +0.176；每日自然 −0.015 |
| L 衰减 | 每 200 tick 一步；**D = 1/(B_T×E) + k×L**（B_T=3，k=0.01），L −= D×step，step = 200/60000 天 |
| 剩余有效天数(约) | **L / D(L,E)**，即按当前衰减率还能维持的天数；L→0 时泌乳结束 |

---

## 典型场景模拟结果（脚本输出摘要）

### 1. 只吃 1 次（第 0 天）

| 药物 | 吸收生效时 L | 耐受 t | 约 0.5 天末 L | 约 1 天末 L | 约 2 天末 L | 剩余有效天数(约) |
|------|----------------|--------|----------------|--------------|--------------|-------------------|
| **Prolactin** | 0.5 | 0.044 | ≈0.42 | ≈0.16 | ≈0 | 吸收后约 **1.5～2 天** 内 L→0 |
| **Lucilactin** | 2.0 | 0.176 | ≈1.83 | ≈1.50 | ≈1.0 | 吸收后约 **6 天** 内 L→0 |

- 药效剩余时间：Prolactin 一针约 **1.5～2 天**；Lucilactin 一针约 **6 天**（与《耐受水池模拟结果》一致）。

### 2. Prolactin 吃 3 次，间隔 0.5 天（服药时刻 0、0.5、1 天）

- 吸收生效时刻：0.25、0.75、1.25 天。
- 第 1 剂后 L=0.5、t=0.044；第 2 剂前 L 已衰减，再 +0.5×E_tol(0.044)≈0.48；第 3 剂再 +0.5×E_tol(0.088)≈0.46。
- 约 **1.25 天末** L≈1.2～1.4、t≈0.13，**剩余有效天数(约)** ≈ 2～3 天。
- 之后不再吃药则 L 按 D(L,t) 衰减，约 **3～4 天** 后 L→0。

### 3. Prolactin 吃 5 次，每天 1 次（服药时刻 0、1、2、3、4 天）

- 每天先日衰减再吸收生效，第 5 次吸收在 4.25 天。
- 约 **4.25 天末**：耐受 t 约 0.18～0.22，L 约 2.5～3.5；**剩余有效天数(约)** ≈ 4～6 天。
- 若此后不再吃药，约 **8～10 天** 内 L→0。

### 4. 同一天吃 3 次 Prolactin（0、0.25、0.5 天）

- 吸收生效在 0.25、0.5、0.75 天，三次间隔短，耐受依次 0.044、0.088、0.132。
- 约 **0.75 天末** L≈1.2～1.3，t≈0.13；**剩余有效天数(约)** ≈ 2～2.5 天。

### 5. Lucilactin 吃 2 次（0、2 天）

- 吸收生效在 0.25、2.25 天。第 2 剂前已有一定衰减，E_tol(0.176)≈0.82，ΔL≈2×0.82=1.64。
- 约 **2.25 天末** L≈2.5～3.0，**剩余有效天数(约)** ≈ 7～9 天。

## 如何运行脚本

在项目或 `记忆库/docs/` 目录下执行：

```bash
python drug_n_doses_simulate.py
```

会输出多组表格：**时间(天) | L | 耐受t | E_tol | 剩余有效天数(约) | 备注**（吸收生效 / 采样）。

可修改 `main()` 中 `dose_times`、`drug_preset`（PROLACTIN / LUCILACTIN）、`max_days`、`sample_interval_days` 做自定义场景。
