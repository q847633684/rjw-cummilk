# 乳池拓扑决策表（rjw-cummilk）

维护者与改码时对照用：当前实现已固定为**虚拟左·右**（无全局拓扑枚举/设置项；旧档中的 `EM.RjwBreastPoolTopologyMode` 存档键将被忽略）。改 `GetBreastPoolSideRows` / `BuildCachedBreastPoolEntries` / 健康 Tab 分列时务必对照本表。

## 当前模式总览（仅虚拟左·右）

| 模式 | 枚举 | 池条数（典型） | `FluidPoolEntry.Key` | `Site` 与 Key |
|------|------|----------------|----------------------|---------------|
| 虚拟左·右 | `VirtualLeftRight` | 每可泌乳**解剖**叶 1 行（每行映射到一对 `_L/_R` 存储键） | 稳定基键由 `MakeStablePoolKey` 生成，实际存储键为 `base_L` / `base_R` | `Site` 为 `BreastLeft` / `BreastRight`，表示虚拟左右槽位；业务状态以 **Key** 为 SSOT |

## UI 左栏 / 右栏含义（健康 Tab、管理面板等）

| 左栏 | 右栏 | 说明 |
|------|------|------|
| 解剖左乳 + `Site==BreastLeft` 的池汇总 | 解剖右乳 + `Site==BreastRight` 的池汇总 | 多叶时为该侧多池求和；存储、满度、R/I 均按 Key 处理 |

## 喷乳反射 R、炎症 I 的键空间

- 持久化字典：`letdownReflexByKey`、`inflammationByKey`（`HediffComp_EqualMilkingLactating`）。
- **`SyncPoolKeyedStateToEntries`**：在 `FluidPoolEntry` 列表变化时，按 **`newEntries[i].Key`** 构造新键集；与旧键集一致则修剪孤儿键，不一致则按旧池奶量加权迁移 R/I。
- **结论**：R / I 与 **`FluidPoolEntry.Key`** 对齐，不依赖旧拓扑枚举；挤单侧只更新命中的键（除非 `GetPoolKeyForBreastHediff` 未命中 → 见下）。

## 开发模式自检（限频日志）

- **`BreastPoolTopologyDiagnostics.MaybeDevWarnAfterEntriesBuilt`**（在 `GetBreastPoolEntries` 与 `CompEquallyMilkable.GetCachedEntries` 重建后调用）  
  - 若某条目 `Site == None` → Warning（常见：HAR/异种体侧向未标注）。
- **`GetPoolKeyForBreastHediff` 未匹配**：`MaybeDevLogPoolKeyLookupMiss`（限频），提示调用方可能拿到 `null`。

## 性能说明

- **`BreastPoolSideRowsCache`**：同一 tick、`BuildBreastListPoolKeySignature` 不变时复用 `GetBreastPoolSideRows` 结果，减轻大殖民地同帧多次列表构建。
