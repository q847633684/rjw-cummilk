# 乳池拓扑决策表（rjw-cummilk）

维护者与改码时对照用：**一种全局设置 `rjwBreastPoolTopologyMode` 决定「几条池、键怎么来、UI 左右栏什么意思」**。改 `GetBreastPoolSideRows` / `BuildCachedBreastPoolEntries` / 健康 Tab 分列时务必对照本表，避免「改一侧逻辑炸多拓扑」。

## 三种模式总览

| 模式 | 枚举 | 池条数（典型） | `FluidPoolEntry.Key` | `Site` 与 Key |
|------|------|----------------|----------------------|---------------|
| 虚拟左·右 | `VirtualLeftRight` | 每可泌乳**解剖**叶 1 条（映射到左/右虚拟槽） | 多为 `BreastLeft` / `BreastRight` 枚举名字符串；多叶同侧时为稳定路径键（`MakeStablePoolKey` 类） | `Site` 表示**虚拟泌乳槽位**（左/右），应与分列一致；多叶时 Key 必须区分叶 |
| 胸位合并 | `RjwChestUnified` | 多叶可合并为 **1 条**（实现为单侧 anatomical 汇总行） | 通常单键或胸级键 | 兼容 legacy：`Site` 常落在 `BreastLeft`；**多条目时**仍可能多条键（少见） |
| 每解剖叶 | `PerAnatomicalLeaf` | **每叶 1 条** | **每叶独立稳定 Key**（与 `SourcePart` 路径一致） | `Site` 可为 `None`（键自定义）；泌乳语义以 **Key** 为 SSOT |

## UI 左栏 / 右栏含义（健康 Tab、管理面板等）

| 模式 | 左栏 | 右栏 | 右栏「是否总可信」 |
|------|------|------|-------------------|
| 虚拟左·右 | 解剖左乳 + `Site==BreastLeft` 的池 | 解剖右 + `Site==BreastRight` 的池 | **多叶同侧**时右栏仍是「该侧多池之和」，单池语义正确 |
| 胸位合并 | 通常整胸汇总视图的「主列」 | 「对侧」或第二汇总列（实现依赖 UI） | **仅 1 池时**简单；**条目数 &gt; 1** 时右栏合计可能误导，**以每条目的 Key 与 `FluidPoolEntry` 为准** |
| 每解剖叶 | 常按 `IsLeft`/叶序折到左列展示 | 右列同理 | 右栏是「展示分组」，**存储与 R/I/满度按 Key**，不依赖「一栏 = 一侧」 |

## 喷乳反射 R、炎症 I 的键空间

- 持久化字典：`letdownReflexByKey`、`inflammationByKey`（`HediffComp_EqualMilkingLactating`）。
- **`SyncPoolKeyedStateToEntries`**：在 `FluidPoolEntry` 列表变化时，按 **`newEntries[i].Key`** 构造新键集；与旧键集一致则修剪孤儿键，不一致则按旧池奶量加权迁移 R/I。
- **结论**：R / I 与 **`FluidPoolEntry.Key`** 对齐，**不是**硬编码仅 `BreastLeft`/`BreastRight`；多叶与稳定键普及后，挤单侧只会更新对应 Key 的 R（除非 `GetPoolKeyForBreastHediff` 未命中 → 见下）。

## 开发模式自检（限频日志）

- **`BreastPoolTopologyDiagnostics.MaybeDevWarnAfterEntriesBuilt`**（在 `GetBreastPoolEntries` 与 `CompEquallyMilkable.GetCachedEntries` 重建后调用）  
  - 虚拟左·右：若某条目 `Site == None` → Warning（常见：HAR/异种体侧向未标注）。  
  - 胸位合并：条目数 **&gt; 1** → Warning（提醒 UI 右栏可能误导）。
- **`GetPoolKeyForBreastHediff` 未匹配**：`MaybeDevLogPoolKeyLookupMiss`（限频），提示调用方可能拿到 `null`。

## 性能说明

- **`BreastPoolSideRowsCache`**：同一 tick、`BuildBreastListPoolKeySignature` 不变时复用 `GetBreastPoolSideRows` 结果，减轻大殖民地同帧多次列表构建。
