# 冲突矩阵与载入策略

用于维护 rjw-cummilk 作为“魔改整合包”时的并存规则：哪些必须互斥、哪些只需排序、哪些可共存。

---

## 1. 分级规则

- **互斥（Hard Conflict）**：同类核心系统被重复接管，可能双 Patch、双 Comp 替换、重复结算。
- **排序（Load Order Sensitive）**：可共存，但需要 `loadAfter/loadBefore` 保证 Def/补丁顺序。
- **可选联动（Optional Integration）**：未安装不影响主功能，安装后启用增量集成。

## 3. 维护准则

- 新增外部兼容前，先判定属于“互斥/排序/可选联动”哪一类，再落到 `About.xml`。
- 命中核心热点（如 `CompMilkable`、`JobGiver`、`SexUtility`、核心 Hediff）的外部模组默认先按“潜在互斥”评估。
- 可选联动必须有运行时 gate（检测 packageId）与空安全降级。
- 互斥项必须在文档与 `incompatibleWith` 保持一致，避免“文档说互斥但实际可同开”。
