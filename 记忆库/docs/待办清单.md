# 待办清单

## 一、本 mod 内「类似乳腺炎」的既有功能（参考）

以下 Hediff 均为**多阶段严重度 + 自然变化（SeverityPerDay）+ 影响玩法**，扩展健康/状态时可对齐设计。

| Hediff | 阶段/严重度 | 自然变化 | 玩法影响 |
|--------|-------------|----------|----------|
| **EM_Mastitis** | 0 / 0.5 / 0.8 三档（轻微堵塞→乳房不适→乳腺炎） | −0.02/日 | 进奶流速倍率 1−Severity×0.5（最低 0.5）；意识 capMods |
| **EM_Prolactin_Tolerance** | 0 / 0.3 / 0.6（small→moderate→high） | −0.015/日 | 药效时长倍率；高耐受可挂肾脏损伤 |
| **EM_Prolactin_Addiction** | satisfied / withdrawal 两档 | −0.015/日 | 戒断时痛感、休息、饥饿、意识、进食 capMods |
| **EM_BreastsEngorged** | 单阶段（有/无，由满度添加移除） | 无（代码按 Fullness 增删） | Moving −0.05、Manipulation −0.03 |
| **Cumpilation_ActiveCowpersGland / ActiveMammaries** | 0 / 0.3 / 0.9（wearing off→altered→fresh） | 负值/日 | 产液倍率（FluidAmountChange） |

其它：EM_Prolactin_High（短期单阶段、快衰减）、EM_DrugLactationBurden（单阶段无衰减）、EM_LactatingGain（由代码动态算 capMods）。

## 二、代码位置与主流程（避免框架乱）

这些功能**没有**单独拆成“健康模块”在别处跑，但**分布不同**：

| 功能 | 代码位置 | 是否在挤奶主流程里 |
|------|----------|--------------------|
| **乳腺炎（炎症触发）** | `MilkRelatedHealthHelper.UpdateInflammationAndTryTriggerMastitis()`，由 `CompEquallyMilkable.UpdateMilkPools()` 每 60 tick 调用 | ✅ 是（和双池进水、R 衰减同一条路径；逻辑在 Helper） |
| **乳腺炎（MTB 触发）** | `MilkRelatedHealthHelper.TryTriggerMastitisFromMtb()`，`CompTick` 里每 2000 tick 调用 | ✅ 是（主流程只调一行；逻辑在 Helper） |
| **乳房胀满** | `MilkRelatedHealthHelper.UpdateBreastsEngorged()`，`CompTick` 里每 tick 调 | ✅ 是（主流程只调一行；逻辑在 Helper） |
| **流速受乳腺炎影响** | `PawnMilkPoolExtensions.GetMilkFlowMultiplierFromConditions()`，在 `UpdateMilkPools` 的流速公式里乘进去 | ✅ 是（主流程只调这一个查询，不塞大段逻辑） |
| **耐受 / 成瘾 / 催乳素 High** | 各自 Hediff + Need + 药物 outcomeDoer、WorldComponent 吸收延迟等 | ❌ 否（原版化学/Need 框架，不在 CompEquallyMilkable 的 tick 里） |
| **Cumpilation 临时体液** | `Cumpilation/` 子目录，HediffComp 与 Patch 独立 | ❌ 否（和泌乳主 tick 分离） |

**结论**：乳腺炎、胀满、炎症的**触发与更新**已抽到 **`MilkRelatedHealthHelper`**（`Source/MilkCum/Milk/Helpers/MilkRelatedHealthHelper.cs`），主流程只调三个入口：`UpdateBreastsEngorged`、`UpdateInflammationAndTryTriggerMastitis`、`TryTriggerMastitisFromMtb`；**效果**（流速惩罚）仍通过 `PawnMilkPoolExtensions.GetMilkFlowMultiplierFromConditions` 读 Hediff。后续扩展健康状态时在 Helper 内新增方法即可，避免 `CompEquallyMilkable` 膨胀。
