# ADR-004：信息卡统计补丁移除（RaceProperties.SpecialDisplayStats）

- **status**: 已采纳
- **date**: 2025-03-04

## Context

本 mod 曾通过 Harmony Postfix 挂接 `RaceProperties.SpecialDisplayStats`，在返回值中追加「产奶类型」「产奶价值」两条 `StatDrawEntry`，以便在小人/动物信息卡中展示。实际运行中与游戏信息卡统计构建流程（StatsReportUtility.StatsToDraw、StatWorker.ShouldShowFor 等）冲突，导致：

- 物品信息卡（泌乳素、人奶、精液等 ThingDef）打开即 NRE 或 ArgumentOutOfRangeException；
- 多次尝试加固（req 守卫、空列表替代 Def 请求、枚举原 result 时 try-catch、移除 displayAllByDefault 等）后，小人/动物信息卡也出现无法打开。

根因与「替换/枚举原版返回值」参与统计列表构建时的重入或列表错位有关，难以在保留该补丁的前提下稳定修复。

## Decision

- **移除**对 `RaceProperties.SpecialDisplayStats` 的整段补丁（`RaceProperties_Patch` 类及其 `GetSpecialDisplayStats`），不再向信息卡注入任何统计条目。
- **禁用 AnimalProductionUtility_Patch 的 Transpiler**：原将 `AnimalProductionUtility.AnimalProductionStats` 迭代器内的 `GetCompProperties<CompProperties_Milkable>` 替换为返回 null 以隐藏原版产奶行；该替换会参与信息卡统计列表构建，易导致 NRE/越界。现 Transpiler 改为直接 `return instructions` 不再做任何替换；产奶类型/价值以主表格列为准。带原版 CompMilkable 的动物可能恢复显示原版产奶行，本 mod 使用的种族无该 Comp 故无影响。
- **HealthCardUtility.GetTooltip(Pawn, BodyPartRecord)** 的乳房池 Postfix 整体用 try-catch 包裹，异常时仅 Log.Warning 不向上抛，防止信息卡健康页悬停时崩溃。
- **EqualMilking.Init()** 中不再设置 `StatCategoryDefOf.AnimalProductivity.displayAllByDefault = true`，避免 Def 信息卡拉取该分类下仅支持 Pawn 的 StatWorker 时 NRE。
- **PawnColumnWorker_MilkType.ClickedIcon** 仍不打开 `Dialog_InfoCard(milkDef)`，改用 `Dialog_MessageBox` 展示描述，避免信息卡内部列表越界；注释中引用本 ADR 说明「本 mod 已不再向信息卡注入统计」。
- **本 mod 的 StatDef 使用 Pawn-only 安全 Worker**：`DrugDurationMultiplier`（PawnGeneral）在 Defs/MilkProductionStats.xml 中设置 `statWorkerClass` 为 `MilkCum.Core.StatWorker_PawnOnlySafe`；该 Worker 在 `req.Pawn == null` 时返回 false，且对 base.ShouldShowFor 做 try-catch，避免 StatsReportUtility.StatsToDraw 枚举时对物品/Def 请求调用默认 StatWorker 导致 NRE 与后续列表越界（见报错修复记录「信息卡第 10 次」条）。
- **全局防护（第 11 次修复）**：对 **RimWorld.StatWorker.ShouldShowFor** 增加 Harmony Prefix（StatWorker_ShouldShowFor_Patch）：当 `req.Pawn == null` 时直接返回 false 并跳过原方法，使**所有** StatDef（含原版与其它 mod）在物品/Def 信息卡下都不会执行可能 NRE 的默认 ShouldShowFor，从根源避免 StatsToDraw 中任一 Worker 抛错导致的列表错位与 ArgumentOutOfRangeException。见报错修复记录「信息卡第 11 次」条。

## Consequences

- 小人/动物信息卡中**不再**显示本 mod 添加的「产奶类型」「产奶价值」两行；产奶相关信息以主表格列为准。
- 物品与小人/动物信息卡均可正常打开，不再因本 mod 导致崩溃。
- 若日后需在信息卡中恢复产奶相关展示，须换用不替换 `SpecialDisplayStats` 返回值、不参与 StatsToDraw 列表构建的其它方式（例如仅在不影响统计列表的 UI 层追加展示）。
