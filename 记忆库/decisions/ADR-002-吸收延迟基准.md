# ADR-002：吸收延迟基准 15000 tick

- **status**: 已采纳
- **date**: 2025-03（根据当前代码）
- **最后与代码核对**: 2025-03

## Context

服药后不应立即挂 Lactating 并进水，需有一段「吸收延迟」再生效；延迟应与代谢率相关（代谢快则延迟短）。

## Decision

- **基准**：`PoolModelConstants.BaseAbsorptionDelayTicks = 15000`（约 0.25 游戏日，因 60000 tick/日）。实际延迟 = 基准 × factor，factor 由代谢率在 [0.25, 2] 区间插值得到（见 Constants 注释：Lerp(1.5, 0.5, InverseLerp(...))），不做除法避免爆炸。
- **实现**：`WorldComponent_EqualMilkingAbsorptionDelay` 管理延迟队列；到点后为 pawn 挂 Lactating 并做 AddFromDrug(Δs) 等，见 Docs 第二节。

## Consequences

- 修改基准需改 `Constants.BaseAbsorptionDelayTicks` 并同步 记忆库/docs/泌乳系统逻辑图 第一节「吸收延迟」描述。
- 与「30 tick 进水」「200 tick 衰减」独立；吸收只影响「何时开始」进水，不影响周期本身。
