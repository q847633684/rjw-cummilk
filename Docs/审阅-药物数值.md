# 药物相关数值审阅结论

审阅范围：`PoolModelConstants`、`MilkCumSettings` 中与泌乳药物、耐受、衰减、吸收延迟相关的常量与公式；与 Docs/参数联动表的一致性及合理性。

---

## 1. 常量与公式汇总

### 1.1 PoolModelConstants（Constants.cs）

| 常量 | 值 | 用途 |
|------|-----|------|
| BaseValueT | 3f | baseline≤0 时药物衰减用的 B_T |
| BaseValueTBirth | 10f | 分娩相关（非药物） |
| EffectiveDrugFactorMin | 0.05f | E_tol 下限，产奶效率最低保留 5% |
| NegativeFeedbackK | 0.01f | 每日衰减 = 1/(B_T×E) + **k×L**；τ≈1/k ≈ 100 游戏日 |
| DoseToLFactor | 1f | ΔL = Δs × C_dose |
| BaseAbsorptionDelayTicks | 15000 | 0.25 游戏日；实际延迟 = 该值 × [0.5~1.5] 倍（按代谢率） |
| LactationEndEpsilon | 1E-5f | L &lt; 此值视为泌乳结束 |

### 1.2 MilkCumSettings 药物/衰减相关

| 设置 | 默认值 | 说明 |
|------|--------|------|
| baselineMilkDurationDays | 5f | 单剂(L≈0.5,E=1)时“剩余天数≈基准天数”反推 B_T |
| birthInducedMilkDurationDays | 30f | 分娩诱导泌乳的基准天数，用于 GetEffectiveBaseValueTForDecayBirth |
| ProlactinToleranceGainPerDose | 0.044f | 每剂对耐受 Hediff 的 Severity 增量，需与 XML 一致 |
| toleranceFlowImpactExponent | 1f | E_tol = (1-E)^exponent；1 为线性 |
| toleranceDynamicMu / Nu | 0.03f / 0.08f | 动态耐受 dE/dt = μ×L − ν×E；无 L 时 E 衰减时间常数 ≈ 1/ν ≈ 12.5 天 |
| allowToleranceAffectMilk | true | 关闭则 E_tol 恒为 1 |

### 1.3 公式

- **有效 B_T（药物）**  
  `GetEffectiveBaseValueTForDecay()`  
  - denom = 0.5/baseline − NegativeFeedbackK×0.5  
  - B_T_eff = 1/denom（denom≤0.01 时返回 100 避免除零）

- **每日衰减**  
  D(L,E) = 1/(B_T×E) + k×L + [炎症项]  
  - 剩余天数 ≈ L / D（稳态近似）

- **耐受系数**  
  E_tol = max(1−toleranceSeverity, 0.05)^exponent  
  - 产奶效率 × E_tol；t 为耐受 Hediff 的 Severity

- **吸收延迟（tick）**  
  delay = BaseAbsorptionDelayTicks × Lerp(1.5, 0.5, InverseLerp(0.25, 2, Clamp(代谢率, 0.25, 2)))  
  - 代谢率 0.25→1.5 倍延迟，2→0.5 倍延迟

## 2. 一致性检查

### 2.1 B_T 与 baseline 天数

设计：单次剂量 L≈0.5、E=1 时，剩余天数 ≈ baselineMilkDurationDays。

- baseline = 5 → denom = 0.5/5 − 0.01×0.5 = 0.1 − 0.005 = 0.095  
- B_T_eff = 1/0.095 ≈ 10.53  
- D = 1/(B_T×E) + k×L = 1/10.53 + 0.01×0.5 ≈ 0.095 + 0.005 = 0.1  
- 剩余 ≈ L/D = 0.5/0.1 = **5 天** ✓

结论：与“基准 5 天”一致。

### 2.2 耐受每剂增量与代码

- `MilkingPatch.DoIngestionOutcome_Postfix` 中：  
  `tBefore = GetProlactinTolerance(pawn) − ProlactinToleranceGainPerDose`  
  即默认认为“本剂在 XML 里已给耐受 +0.044”，用药前 t 要减掉该值再算 E_tol。
- 注释已说明：**改 XML 中耐受 Hediff 的 severity 增量时，需同步改 ProlactinToleranceGainPerDose**。

结论：逻辑一致；需确保 Def 中泌乳药物对耐受的 severity 增量为 0.044（或与 ProlactinToleranceGainPerDose 一致）。

### 2.3 DoseToLFactor 与 Severity

- `AddFromDrug`：ΔL = deltaSeverity × DoseToLFactor；deltaSeverity 为耐受后、种族倍率后的“有效 severity”。
- 原版 Lactating 的 severity 与 L 同步（AddToLAndSeverity），DoseToLFactor=1 表示 ΔL = Δs 一一对应。

结论：与当前“L 与 Severity 同步”设计一致。

### 2.4 ProlactinToleranceGainPerDose 未存档

- `ProlactinToleranceGainPerDose` 未在 `ExposeData` 中 Scribe，存档不保存。
- 设计上作为“与 XML 同剂叠加一致”的常量，不暴露给玩家亦可；若将来在 mod 设置中暴露该值，需：  
  (1) 在 ExposeData 中 Scribe，  
  (2) 在文档/注释中注明“若修改需与 XML 中耐受 Hediff 的 severity 增量一致”。

## 3. 合理性评估

| 项 | 评估 |
|----|------|
| baseline 5 天 / 分娩 30 天 | 单剂时长与“分娩诱导更长”的直觉一致，数值可调且已存档。 |
| NegativeFeedbackK = 0.01 | τ≈100 天，高 L 时衰减略加快，避免无限堆 L，范围合理。 |
| EffectiveDrugFactorMin = 0.05 | 高耐受时仍保留 5% 产奶，避免完全无效。 |
| 吸收延迟 0.125~0.375 游戏日 | 约 3~9 小时，符合“服药后一段时间生效”的设定。 |
| 动态耐受 μ=0.03, ν=0.08 | 无药时 E 约 12.5 天半衰期；有 L 时 E 上升，平衡可接受。 |
| 0.044/剂 | 约 23 剂达 t≈1（E_tol 接近下限），需与 XML 和玩法预期一致。 |

## 4. 建议（可选）

1. **文档**  
   在参数联动表或规格中明确写出：  
   - “单剂 L≈0.5、E=1 时剩余天数 = baselineMilkDurationDays”的推导；  
   - ProlactinToleranceGainPerDose 与 Def 中耐受 severity 增量的对应关系。

2. **注释**  
   - 在 `GetEffectiveBaseValueTForDecay` 旁保留或补一句：  
     “单剂 L=0.5、E=1 时剩余天数 ≈ baseline 天（由 denom = 0.5/baseline − k×0.5 反推）”。  
   - 若 Def 中耐受 severity 增量写死为 0.044，可在 ProlactinToleranceGainPerDose 的注释中注明“与 Def xxx 一致”。

3. **ProlactinToleranceGainPerDose 存档**  
   仅当将来在设置中暴露该值时，再在 ExposeData 中增加 Scribe，并同步更新文档。

---

## 5. 结论

- **数值与公式**：B_T 与 baseline/birth 天数、每日衰减 D(L,E)、耐受系数 E_tol、吸收延迟公式均自洽，且与代码实现一致。  
- **与 Docs/参数联动**：设计意图（基准天数、单剂对应、耐受与 XML 一致）在注释中已有体现，建议在 Docs 中补一句“药物数值审阅结论”的引用或摘要。  
- **未发现错误**；上述建议为文档与可维护性增强，非必改。
