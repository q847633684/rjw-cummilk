## 十、差异汇总与建议

### 1. 必须修正（与文档明确不一致）

- **preferLeft 男左女右**  
  - **文档**：两侧水位相同时 `preferLeft = (Pawn.gender == Gender.Male)`（男左女右）。  
  - **代码**：`CompEquallyMilkable.DrainForConsume` 中 `bool preferLeft = false;` 写死。  
  - **建议**：将 `preferLeft` 改为 `(parent is Pawn p && p.gender == Gender.Male)`，使吸奶/挤奶选侧与文档一致。

### 2. 需产品/设计确认（文档与实现二选一）

- **挤奶（Gathered）范围**  
  - **文档**：仅排第一对的一侧，不跨对。  
  - **代码**：Gathered 使用 DrainForConsume(amountToTake)，会跨对取满需求量。  
  - **建议**：  
    - 若设计为「每次挤奶只动第一对的一侧」：在 Gathered 中改为仅从第一对取一侧（扣量上限为该侧当前水量），并产出对应奶量；preferLeft 同上。  
    - 若设计为「每次挤奶可跨对取满」：在《泌乳系统逻辑图》第五节中修改挤奶描述为「与吸奶共用 DrainForConsume，按对顺序取满当次需求量」。

### 3. 文档可补充的扩展

- **E_tol 指数与开关**：代码中 E_tol = [max(1−t, 0.05)]^exponent（toleranceFlowImpactExponent），且 allowToleranceAffectMilk 可关闭。建议在逻辑图「符号与概念」或「公式与常数」中补充说明。

## 十一、审计结论

- **一致**：L/双池/耐受/进水/衰减/常数、B_T 反推、UI、吃药路径、心情与 Hediff、满池撑大与溢出、单乳处理等与文档一致。  
- **差异**：  
  1. **preferLeft** 未按性别实现，建议按文档改为男左女右。  
  2. **挤奶范围** 文档写「仅第一对的一侧」，代码为跨对取满，需产品确认后要么改代码要么改文档。  
- **扩展**：E_tol 指数与开关可在文档中补充。

完成上述修正与文档更新后，可实现与《泌乳系统逻辑图》的完全对齐。
