# 泌乳系统 · 全部说明（rjw-cummilk）

本文档汇总本 Mod 的泌乳系统设计、数据流、公式与涉及文件，便于维护与扩展。

---

## 1. 概述与设计目标

- **定位**：在 RimWorld + RJW 下，为带「乳房」部位的 Pawn 提供**平等挤奶**（Equal Milking）：多侧/多对乳房共享一套池模型，按对进水、按侧扣量，支持吸奶/手挤/机器挤、营养消耗与回缩吸收、满池溢出与信件、RJW 体型同步。
- **核心设计**：
  - **L 驱动**：泌乳量 L（药物分量 + 分娩分量）驱动每日进池流速；L 衰减则泌乳结束。
  - **双池模型**：每对乳房为「左+右」两个池侧；先填满基础容量，仅当两侧都满后才允许撑大（Stretch），超出部分算溢出。
  - **1 池单位 = 1 营养**：产奶消耗营养、回缩吸收加回营养，Need_Food/Need_Energy 补丁与回缩吸收均按此对等。
- **前提**：仅当 Pawn 存在 RJW 乳房 Hediff（`GetBreastListOrEmpty()` 非空）且 Mod 设置开启 RJW 乳房容量时，才创建乳池、进水、挤奶；无乳房则返回空列表、不建池。  
  - 说明：**乳房列表由 RJW 提供**（`GetBreastList()`），本 Mod 用 `GetBreastListOrEmpty()` 做空安全封装；**「无乳房时不建池、不进水、不挤奶」由本 Mod 保证**——在 `GetBreastPoolEntries()` 等处若列表为空则返回空，后续池逻辑自然不执行，不能依赖 RJW 代为处理。

## 2. 核心概念

### 2.1 泌乳量 L（CurrentLactationAmount）

- **定义**：`L = lactationAmountFromDrug + lactationAmountFromBirth`（在 `HediffComp_EqualMilkingLactating` 内）。
- **药物分量**：吃药/打针时通过 `AddFromDrug(severity)` 增加，按 `PoolModelConstants.DoseToLFactor` 与耐受 E 折算；每日衰减 = 1/(B_T×E) + k×L（负反馈）。
- **分娩分量**：分娩时增加 `BaseValueTBirth`，单独衰减基准 B_T_birth。
- **永久泌乳**：基因 `EM_Permanent_Lactation` 或设置「雌性成年动物始终泌乳」时，L 不衰减，Severity 维持 ≥0.9999 或 1。
- **流速用 L**：`EffectiveLactationAmountForFlow = min(L, lactationLevelCap)`（lactationLevelCap>0 时封顶），用于驱动力与进池流速计算。

### 2.2 驱动力与进池流速

- **驱动力**：`D = GetEffectiveDrive(EffectiveL)`，低 L 近似线性、高 L 饱和（MilkCumSettings）。
- **每日基础流速**：`basePerDay = D × hungerFactor × condFactor × raceFlow`（condFactor = 健康/炎症/乳腺炎等，raceFlow = 种族流速倍率）。
- **每 60 tick 进池量**：`flowPerTickScale = max(0, basePerDay / TicksPerGameDay * Interval60Ticks)`；单侧实际流速再乘：该侧 FlowMultiplier、Conditions、压力因子、喷乳反射 R、Density。

### 2.3 池侧与 Key 约定

- **FluidPoolEntry**：`Key`（稳定字符串）、`Capacity`、`FlowMultiplier`、`IsLeft`、`PairIndex`、`Density`。
- **Key 生成**：`GetBreastPoolEntries` 中按 `baseKey + "_" + i + "_L"` / `"_R"`，baseKey 为部位 defName 或 hediff.defName，i 为乳房 Hediff 在 GetBreastList 中的序号；同一对左右共享同一 `PairIndex`。
- **breastFullness**：`Dictionary<string, float>`，key 与 Entry.Key 一致；存读档用 `Scribe_Collections.Look(..., "BreastFullness", LookMode.Value, LookMode.Value)`，并保留 keys/vals 兼容旧档。

### 2.4 双池模型（每对）

- **基础容量**：左右各 `Capacity`（来自 RJW Hediff Severity × 系数）。
- **撑大容量**：`stretchCap = Capacity * StretchCapFactor`（PoolModelConstants.StretchCapFactor = 1.2）。
- **进水规则**（FluidPoolState.TickGrowth）：
  - 阶段一：只填到基础容量；一侧满后富余流速可加到另一侧。
  - 阶段二：仅当**阶段一后的** newLeft ≥ leftBaseCap-ε 且 newRight ≥ rightBaseCap-ε 时，才把剩余流速填入撑大区；再超出部分为溢出。
- **回缩**：超出基础容量的部分每 60 tick 乘 `shrinkFactor = Clamp01((1 - ShrinkPerStep) * healthPercent)`，向基础容量收敛；回缩部分按 `NutritionPerPoolUnit` 与 `reabsorbNutritionEfficiency` 折算为饱食度/能量加回（GetReabsorbedNutritionPerDay）。

### 2.5 满池与溢出

- **满池判定**：某对「左右均 ≥ FullnessThresholdFactor×baseCap」或单侧「newFullness ≥ FullnessThresholdFactor×cap」时计为满；`ticksFullPool` 在本步任一侧/对满时 +60，否则衰减（-120）。
- **溢出**：每 tick 超出撑大容量的部分累加到 `overflowAccumulator`；达到 `OverflowFilthThreshold` 时生成地面污物（Def 可配置），并扣减累计量；`overflowAccumulator` 有硬上限 `OverflowAccumulatorCap = 100f`。
- **溢出状态**：某侧发生过溢出且当前水位仍高于基础容量时，该侧进入「溢出状态」：本步不再进水，只做回缩，直到水位 ≤ 基础容量再清除标记。

## 4. 组件与职责

### 4.1 CompEquallyMilkable（ThingComp，挂于 Pawn）

- **分片**：主类 + PoolStorage + Inflow + Drain + Milking。
- **PoolStorage**：`GetCachedEntries()` / `GetCachedPairGroups()`、脏标记（SetEntriesCacheDirty、EnsureEntriesCacheDirtyIfBreastCountChanged）、`GetFullnessForKey`、按对/按侧汇总、ClearPools、SetFullness；选侧规则 ADR-003（同对先扣较满一侧，相等先左）。
- **Inflow**：`UpdateMilkPools()`（每 60 tick 在 CompTick 内、cachedActive 时调用）：营养消耗、进池、回缩、溢出累计、满池信件、回缩吸收加回、CachedFlowPerDayForDisplay / P/L/C 显示缓存；`HandleOverflow`、`TrySendFullPoolLetter`。
- **Drain**：`DrainForConsume(amount, drainedKeys, singleSideOnly)`、`DrainForConsumeByRates`（机器挤奶按侧速率）、`GetFirstDrainSideIndex`（吸奶时选「第一个被扣」的侧）。
- **Milking**：与产奶 Job、Gather 回调、LastGatheredTick、MilkSettings、allowedSucklers/allowedConsumers 等相关。
- **Gizmo**：DevMode 且 Active 时显示「池 P= L= C=」及 overflow/满度/流/天/回缩吸收（CompGetGizmosExtra）。

### 4.2 HediffWithComps_MilkCumLactating（Lactating Hediff）

- 阶段由 C# 动态生成（GenStages），括号内「永久」由 HediffComp 判定；CurStageIndex 依赖 Fullness 与 maxFullness 的满池状态。
- 挤奶/吸奶回调：OnGathered、OnGathered(fullness)、OnGatheredLetdownByKeys(drainedKeys)（按被扣量的侧加喷乳反射）；炎症模型下 AddMilkingLStimulus。

### 4.3 HediffComp_EqualMilkingLactating

- L 两分量、喷乳反射 R 按 key 存储、炎症 I、耐受 E、永久泌乳判定、RemainingDays、EffectiveLactationAmountForFlow。
- 药物/分娩进水：AddFromDrug、分娩时加 L；每日衰减与负反馈、UpdateToleranceDynamic。
- 喷乳反射：GetLetdownReflexFlowMultiplier(key)、AddLetdownReflexStimulus(key)、DecayLetdownReflex。
- 炎症：UpdateInflammation、TryTriggerMastitisFromInflammation；MTB 乳腺炎由 MilkRelatedHealthHelper.TryTriggerMastitisFromMtb 每 2000 tick 判定。

## 5. 进水与回缩（UpdateMilkPools 概要）

1. **前置**：LactatingHediffComp 存在且 RemainingDays>0；L>0 且 hungerFactor>0；否则清空显示缓存并 return。
2. **营养消耗**：若 Fullness < maxFullness - capacityAdaptation，按 ExtraNutritionPerDay × GetNutritionFactorForExtra() × Interval60PerDay 扣食物/能量。
3. **进池**：按 GetCachedPairGroups 遍历每对或单侧；每对调用 `FluidPoolState.TickGrowth(flowLeft, flowRight, BreastPairCapacities)`，单侧调用 `SingleBreastTickGrowth`；flow 已乘压力/喷乳/状态，且入口 clamp（flowPerTickScale≥0，shrinkFactor∈[0,1]，flowPerTick≥0）。
4. **回缩与溢出**：若某侧 newFullness > baseCap，超出部分一部分回缩（×shrinkFactor 保留在池），其余为回缩吸收；overflow 累加到 overflowTotal，并标记 overflowTriggeredByKey；最后 HandleOverflow(overflowTotal)，overflowAccumulator 上限 100f。
5. **ticksFullPool**：本步任一对满或任一侧满则 +60，否则 max(0, ticksFullPool-120)。
6. **显示缓存**：CachedFlowPerDayForDisplay、CachedPressureForDisplay、CachedLetdownForDisplay、CachedConditionsForDisplay；cachedReabsorbedNutritionPerDay、hadShrinkLastStep。
7. **健康**：UpdateInflammationAndTryTriggerMastitis；UpdateBreastsEngorged（满池加胀满 hediff，<90% 移除）；TryTriggerMastitisFromMtb 在 CompTick 每 2000 tick 调用。

## 6. 扣量与选侧（Drain）

- **DrainForConsume(amount, drainedKeys, singleSideOnly)**：
  - singleSideOnly=true（吸奶）：只从 GetFirstDrainSideIndex() 指定的那一侧扣，最多 amount，drainedKeys 记录被扣的 key，用于喷乳反射按侧刺激。
  - singleSideOnly=false（手挤）：按「对」按满度降序，同对内先扣较满一侧、相等先左，直到扣满 amount；浮点余量用 FloatDustEpsilon 吸收。
- **DrainForConsumeByRates**：机器挤奶按每侧速率并行扣量，总扣量不超过 remainingCap，与 GetCachedEntries 顺序一致。

## 7. 健康与事件

- **MilkRelatedHealthHelper**：
  - EnsureLactatingHediffFromConditions：基因永久泌乳 / 非人形不可挤奶移除 / 动物始终泌乳；增删 Lactating 后调用 `SetEntriesCacheDirty()`。
  - UpdateBreastsEngorged：满池加 EM_BreastsEngorged，<90% 移除。
  - UpdateInflammationAndTryTriggerMastitis：四层炎症 I 更新，I>I_crit 触发乳腺炎。
  - TryTriggerMastitisFromMtb：长时间满池、卫生差、躯干伤等 MTB 判定，添加或加重 EM_Mastitis。

- **GetMilkFlowMultiplierFromConditions**：健康/乳腺炎/炎症等对流速与回缩的倍率（供压力/回缩公式使用）。

- **乳房相关病例（Hediff，挂于胸部/乳房部位）**：
  | Def | 中文名 | 说明 | 触发/移除 |
  |-----|--------|------|-----------|
  | **EM_Mastitis** | 乳腺炎 | 对应医学乳汁淤积与哺乳期乳腺炎；三阶段（堵奶/乳汁淤积→乳腺炎（炎症）→乳腺炎（重度）），高 severity 降意识。 | 触发：长时间满池、卫生差、躯干/乳房受伤（MTB 每 2000 tick）；卫生差+淤积或损伤时 MTB 再÷mastitisInfectionRiskFactor；或四层炎症 I>I_crit。缓解：severity 每日 -0.02；池满度&lt;30% 时 HediffComp_MastitisDrainageRelief 额外每日 -0.01。 |
  | **EM_BreastsEngorged** | 乳房胀满 | 奶水过满，移动 -5%、操纵 -3%。 | 满池（≥95% maxFullness）时添加；<90% 时移除。 |
  | **EM_DrugLactationBurden** | 药物泌乳负担 | 长期药物泌乳的轻微身体负担，移动 -2%、操纵 -1%。 | 由药物/成瘾与泌乳状态逻辑添加与移除。 |
  | **EM_LactatingGain** | 泌乳增益 | 药物泌乳时设置开启后的正面加成（意识/操纵/移动），非病例。 | 条件满足时 CompEquallyMilkable 添加，否则移除。 |
  | **EM_AbsorptionDelay** | 吸收延迟 | 泌乳药物尚未完全吸收，泌乳即将生效。 | ScheduleLactating 时添加，ApplyDelayedLactating 时移除。 |
  | **Lactating**（原版） | 泌乳 | 泌乳状态，挂于 GetBreastOrChestPart()。 | 基因/设置/动物始终泌乳/药物等由 EnsureLactatingHediffFromConditions 维护。 |
  | **Lactating_Drug / Lactating_Permanent / Heavy_Lactating_Permanent** | 药物泌乳 / 永久泌乳 / 重度永久泌乳 | 兼容用标签型 hediff，非病例。 | 依原 rjw-milkable-colonists 系逻辑。 |

  其他与泌乳相关但未必挂在乳房部位的 Hediff：**EM_Prolactin_Tolerance**（催乳素耐受）、**EM_Prolactin_High**（催乳素兴奋）、**EM_Prolactin_Addiction**（催乳素成瘾），见 Defs/HediffDefs.xml。

- **医学上乳腺炎与本 Mod 的对照与建议**：
  - **医学定义**：乳腺炎（mastitis）= 乳腺组织**炎症**，可伴或不伴**感染**。哺乳期乳腺炎（lactational mastitis）通常 = **乳汁淤积（milk stasis）** + **细菌经破损皮肤/乳头侵入**；常见诱因：排空不全/堵奶、乳头皲裂、卫生差、过度胀满。
  - **本 Mod 现状（已贴近医学）**：单一 Hediff **EM_Mastitis**（标签「乳腺炎」），三阶段对应「堵奶/乳汁淤积→乳腺炎（炎症）→乳腺炎（重度）」；描述中写明对应医学乳汁淤积与哺乳期乳腺炎、诱因与治疗要点。触发为 **(1) MTB**：长时间满池、卫生差、躯干/乳房受伤；**卫生差且（淤积或损伤）时** MTB 再除以 `mastitisInfectionRiskFactor`（感染风险）。(2) **四层炎症**：I>I_crit 时添加/加重。恢复：severity 每日 -0.02 + **排空即缓解**（池满度 &lt; 阈值时 HediffComp_MastitisDrainageRelief 额外每日 -0.01）。
  - **主要区别**：医学区分「非感染性堵奶/炎症」与「感染性乳腺炎」；本 Mod 用一条 Hediff 覆盖从堵奶到乳腺炎，用**卫生因子 + 感染风险系数**近似「感染风险」，未做病原体或抗生素逻辑。
  - **已实现（更贴近医学）**：  
    1. **命名/阶段**：Def 与阶段 label 改为「堵奶/乳汁淤积」「乳腺炎（炎症）」「乳腺炎（重度）」；description 说明对应医学、诱因与排空治疗。  
    2. **感染路径**：`mastitisInfectionRiskFactor`（默认 1.2）；当 badHygiene && (longFull \|\| torsoInjury) 时 MTB 再除以此系数。  
    3. **排空即缓解**：`HediffComp_MastitisDrainageRelief`，池满度 &lt; `MastitisDrainageReliefFullnessThreshold`（0.3）时每日额外 severity 衰减 `MastitisDrainageReliefSeverityPerDay`（0.01）；常量在 PoolModelConstants。  
    4. **设置**：MilkRiskSettings 新增 `mastitisInfectionRiskFactor`，存档键 EM.MastitisInfectionRiskFactor。

## 8. RJW 集成（体型同步）

- **RJWLactatingBreastSizeGameComponent**：
  - BreastBaseSeverity：key = `"{LoadID}:{breastIndex}"`，存施加前 RJW 乳房 Severity；读档用 PendingRestore 列表解析回字典。

  - CleanupDeadPawns：用 `map.mapPawns.AllPawns` 构建有效 loadID 集合，移除不在任何地图的 pawn 的 key，防止泄漏；含尸体以利读档恢复。

- **SyncRJWBreastSeverityFromPool**：池满度与 L 驱动目标 Severity = baseSev + rjwLactatingSeverityBonus×t_L + rjwLactatingStretchSeverityBonus×t_pool；BreastBaseSeverity 无 key 时用当前 Hediff Severity 写入。
