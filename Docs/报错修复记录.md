## 重复修复 · 根因分析

- **同一处**：`HediffSet_Remove_Patch` 中「如何拿到被移除 hediff 所属的 Pawn」。
- **根因**：Patch 的是 RimWorld 原版类型（`HediffSet`/`Pawn_HealthTracker`），未直接引用游戏程序集时无法确认其公开成员名（`Remove`/`pawn`/`Pawn` 均猜错）。
- **最终方案**：不依赖 `Pawn_HealthTracker` 的成员，直接使用 Postfix 已有的 `hediff` 参数，用 **`hediff.pawn`** 取所属 Pawn，避免依赖原版内部 API 命名。

- **日期**：2025-03-01
- **报错摘要**：MilkCum Patch failed for ITab_Pawn_Feeding_Patch — Patching exception in DrawRow(baby, RectDivider& row, feeder, mother, father, surrogate)
- **位置**：`Source/MilkCum/Milk/HarmonyPatches/Breastfeed.cs`，对 `ITab_Pawn_Feeding.DrawRow` 的 Transpiler
- **修复**：DrawRow 拆到单独 Patch 类，用 `TargetMethod()` 仅当方法为旧版 3 参数 (baby, row, feeder) 时才返回方法并打补丁；当前 1.6 为 6 参数 (baby, RectDivider&, feeder, mother, father, surrogate) 时返回 null 不 patch，避免运行时 Patching exception，喂食标签仍为原版显示。
- **重复？**：否
