# 泌乳池优化 · 全部实现方案

按依赖与风险排序，逐项实施。每项含：目标、涉及文件、设计要点、步骤。

---

## 阶段一：低风险 / 无依赖（先做）

### 1. IsCachedFlowValid 里 TicksGame 缓存 ✅ 已完成
- **目标**：CompTick 每 60 tick 把 `Find.TickManager.TicksGame` 写入字段，IsCachedFlowValid 用该字段判断，减少重复调用。
- **涉及**：`CompEquallyMilkable.cs`（新增字段 `cachedTicksGameForFlow`，在 60-tick 块内赋值；IsCachedFlowValid 用 `(cachedTicksGameForFlow - CachedFlowTick) <= 60`）。
- **步骤**：加字段 → 在 UpdateMilkPools 被调用的同一 60-tick 分支末尾赋 `cachedTicksGameForFlow = Find.TickManager.TicksGame` → IsCachedFlowValid 改为用 `cachedTicksGameForFlow`。

### 2. EnsureLactatingHediffFromConditions 抽到 MilkRelatedHealthHelper ✅ 已完成
- **目标**：把 CompEquallyMilkable 内 EnsureLactatingHediffFromConditions 的逻辑抽成静态方法，减少 Comp 体积、便于复用与测试。
- **涉及**：`MilkRelatedHealthHelper.cs`（新增 `EnsureLactatingHediffFromConditions(Pawn pawn)`），`CompEquallyMilkable.cs`（改为调用 helper）。
- **步骤**：在 MilkRelatedHealthHelper 中实现静态方法（基因/物种/设置分支，GetOrAddHediff/RemoveHediff）→ Comp 内改为一行调用并删原实现。

### 3. fullPoolLetterCooldownDays 的 UI ✅ 已完成
- **目标**：在 Mod 设置里为「满池信件冷却天数」加滑块与 Keyed。
- **涉及**：`Widget_AdvancedSettings.cs`，`Languages/English/Keyed/lang.xml`，`Languages/ChineseSimplified/Keyed/lang.xml`。
- **步骤**：在 DrawOverflowBlock 中增加 enableFullPoolLetter 复选框、fullPoolLetterCooldownDays 滑块（0.5~7 天）；Keyed：EM.EnableFullPoolLetter、EM.EnableFullPoolLetterDesc、EM.FullPoolLetterCooldownDays、EM.FullPoolLetterCooldownDaysDesc。

### 4. log 用 StringBuilder / 仅 debug 开启 ✅ 已完成
- **目标**：PoolTickLog 拼接用 StringBuilder；可选：仅 DEBUG 或开发模式时输出详细 log。
- **涉及**：`CompEquallyMilkable.Inflow.cs`（两处 PoolTickLog 块），可选 `MilkCumSettings.cs`（如加「仅 debug 输出」开关）。
- **步骤**：两处 log 块内用 StringBuilder 拼接再一次性 PoolTickLog(sb.ToString())；输出已由 MilkCumSettings.PoolTickLog 内 Prefs.DevMode && lactationPoolTickLog 控制，无需额外 #if DEBUG。

## 阶段二：中等风险 / 需设计

### 5. GetCachedEntries 脏标记 + 300–600 tick 刷新 ✅ 已完成
- **目标**：cachedEntries 不用固定 60 tick 失效，改为「脏标记 + 最长 300 或 600 tick 刷新」：hediff 变化或乳房列表变化时设脏，GetCachedEntries 若脏或超间隔则重建。
- **涉及**：`CompEquallyMilkable.PoolStorage.cs`（GetCachedEntries、GetCachedPairGroups、GetCachedEntriesIfValid、ClearPools），`CompEquallyMilkable.cs`（CompTick 120-tick 内调用 EnsureEntriesCacheDirtyIfBreastCountChanged）。
- **实现**：`cachedEntriesDirty`、`lastBreastListCount`、`EntriesCacheMaxTicks = 300`；GetCachedEntries 在非脏且 (now - cachedEntriesTick) ≤ MaxTicks 时直接返回缓存，否则重建并清脏、更新 lastBreastListCount；每 120 tick 调用 EnsureEntriesCacheDirtyIfBreastCountChanged() 比较 GetBreastListOrEmpty().Count 与 lastBreastListCount 设脏；ClearPools 时设脏并 lastBreastListCount = -1。

### 6. MilkSettings 读档后立即初始化 ✅ 已完成
- **目标**：PostLoadInit 后对含 CompEquallyMilkable 的 Pawn 调用一次 GetDefaultMilkSetting() 写入 milkSettings，避免首次访问 getter 时才赋值。
- **涉及**：`CompEquallyMilkable.cs`（PostExposeData 的 PostLoadInit 块）。
- **实现**：在 PostLoadInit 块内 EnsureSaveCompatAllowedLists() 之后增加：若 milkSettings == null 且 parent is Pawn，则 milkSettings = pawn.GetDefaultMilkSetting()。

### 7. allowedSucklers/Consumers 事件驱动清理 ✅ 已完成
- **目标**：将 allowedSucklers/allowedConsumers 的清理集中到 GameComponent，替代每个 Comp 每 2500 tick 的 RemoveAll。
- **涉及**：`CompEquallyMilkable.cs`（新增 internal CleanupAllowedLists()，移除 CompTick 内 2500 tick 块），`RJWLactatingBreastSizeGameComponent`（每 2500 tick 遍历 Find.Maps 所有 Pawn，对 CompEquallyMilkable 调用 CleanupAllowedLists()）。
- **实现**：逻辑集中到现有 RJWLactatingBreastSizeGameComponent.GameComponentTick()，当 TicksGame % 2500 == 0 时遍历所有地图 AllPawnsSpawned，对每个 pawn.CompEquallyMilkable() 调用 CleanupAllowedLists()；CompTick 中删除 2500 tick 分支。

### 8. healthPercent × 炎症/乳腺炎因子
- **目标**：回缩时用的 healthPercent 再乘一个「炎症/乳腺炎」因子（如 Mastitis 存在则 0.5~1），使有炎症时回缩更慢。
- **涉及**：`CompEquallyMilkable.Inflow.cs`（shrinkFactor 计算处），`MilkRelatedHealthHelper` 或已有 GetMilkFlowMultiplierFromConditions。
- **设计**：`inflammationFactor = Pawn.GetMilkFlowMultiplierFromConditions()`（或同源），shrinkFactor = (1f - ShrinkPerStep) * healthPercent * inflammationFactor。
- **步骤**：在计算 shrinkFactor 处取 `float inflammationFactor = Pawn != null ? Pawn.GetMilkFlowMultiplierFromConditions() : 1f`，再 shrinkFactor *= inflammationFactor。

## 阶段三：高影响 / 大改（最后做）

### 9. maxFullness 改名为 maxPoolCapacity
- **目标**：语义更清晰，避免与「满度」混淆。
- **涉及**：全局替换 `maxFullness` → `maxPoolCapacity`（含 Scribe 的 key 若希望兼容可保留旧 key 读一次）。
- **步骤**：全项目替换 maxFullness 为 maxPoolCapacity；Scribe 的 "PoolMaxFullness" 可保留读旧档或改为 "PoolMaxPoolCapacity" 并做向后兼容读。

### 10. breastFullness 用 Hediff 或 BodyPartRecord 当 key
- **目标**：key 从 string 改为可稳定标识「哪一侧」的引用或结构；Hediff 存读档复杂，BodyPartRecord 可能因手术变化；若仍用 string 则保持现状也可。
- **设计**：更稳妥的是保留 string key，在文档中明确「key 的生成规则」与「与 GetBreastPoolEntries 一致」；若强行改为 Hediff/BodyPartRecord，需要：breastFullness 类型改为 Dictionary<Hediff, float> 或 Dictionary<BodyPartRecord, float>，但一对乳房对应一个 Hediff 两个 key（左/右），所以更自然的是 Dictionary<(Hediff, bool isLeft), float> 或仍用 string（来自 Hediff+index+_L/_R）。建议此项标为「不实施」或「仅文档化 key 约定」，除非有强烈需求。
- **步骤**：若做，需：定义 Key 类型（如 struct BreastPoolKey { Hediff h; bool isLeft; } 并实现 Equals/GetHashCode、Scribe），所有 breastFullness 的读写与 Scribe 改为新类型；Drain/Inflow/GetFullnessForKey 等全部适配。工作量大且易引入 bug。

### 11. Gizmo / float menu 显示 pressure、letdown、conditions
- **目标**：调试或平衡时在 pawn 的 Gizmo 或右键 float menu 中显示当前 CachedPressureForDisplay、CachedLetdownForDisplay、CachedConditionsForDisplay 等。
- **涉及**：新增 Gizmo 或 CompEquallyMilkable 的 CompGetGizmosExtra / 或通过 Pawn 的 FloatMenu/Inspect 扩展。
- **步骤**：在 CompEquallyMilkable 中重写 CompGetGizmosExtra（或等价），当 Active 时添加一个 Command_Action 或 Info Gizmo，Label 为 "池: P=... L=... C=..."，仅开发/调试时显示可用 Prefs.DevMode 或设置项控制。

## 优先修复 5 项（体感/稳定性，建议顺序）

### A1. FluidPoolState.TickGrowth 的 bothAtBase 改用阶段一后的 newLeft/newRight ✅ 已完成
- **目标**：避免「本 tick 刚好补满但不进入 stretch」导致撑大难、溢出多；玩家体感「奶量满了但不涨体型」。
- **实现**：`FluidPoolState.cs` 中 bothAtBase 改为 `newLeft >= leftBaseCap - eps && newRight >= rightBaseCap - eps`；TickGrowth 改为纯计算、返回 `(newLeft, newRight, overflow)` 不修改内部状态；Inflow 用返回值做回缩并写 breastFullness。

### A2. UpdateMilkPools 进水与回缩合并为同一遍历 ✅ 已完成
- **目标**：消除 1 tick 延迟，满池当轮即回缩。
- **实现**：`FluidPoolState.TickGrowth` 改为返回 `(float, float, float)` 不写 LeftFullness/RightFullness；Inflow 内 `var (leftNew, rightNew, overflow) = pairPool.TickGrowth(...)` 后同一循环内做回缩并写 breastFullness；单乳分支不变。

### A3. RJWLactatingBreastSizeGameComponent 的 Dictionary key 改为 string ✅ 已完成
- **目标**：Hediff 被替换/重新生成时 key 失效导致泄漏与恢复失败。
- **实现**：`BreastBaseSeverity` 改为 `Dictionary<string, float>`，key = `$"{pawn.GetUniqueLoadID()}:{breastIndex}"`；新增 `KeyFor(Pawn, int)`、`ParseKey(string)`、`HasAnyKeyForPawn`；ExposeData/ResolvePending/ApplyPermanentBreastGain/SyncRJWBreastSeverityFromPool/ApplyOrRestoreBreastSeverity/CleanupDeadPawns 全部改用 string key；补 `using UnityEngine` 修复 Mathf。

### A4. 废除 leftFullness/rightFullness，只保留 breastFullness + key ✅ 现状已符合
- **目标**：多乳/不对称时显示与计算同步。
- **说明**：Comp 层 LeftFullness/RightFullness 已由 `GetLeftOrRightFullness` 从 breastFullness 按 entries 汇总，无独立 left/right 存储；FluidPoolState 为 per-pair 临时计算保留内部左右字段。无需改代码。

### A5. ticksFullPool 统一 + 上限保护 ✅ 已完成
- **目标**：多对乳房时计数准确，满池信件/乳腺炎触发一致；防止溢出累计爆增。
- **实现**：在 UpdateMilkPools 循环内设 `anyFullThisStep`，每对满（左右均 ≥ 0.95×baseCap）或单乳满（newFullness ≥ 0.95×cap）时置 true；循环外若 anyFullThisStep 则 ticksFullPool += 60 否则衰减；HandleOverflow 末尾 `overflowAccumulator = Mathf.Min(overflowAccumulator, OverflowAccumulatorCap)`（硬上限 100f）。

## 中期可优化（逐项实现）

### B1. 乳房 hediff 变化/增删时标记 entries 缓存脏
- **目标**：GetCachedEntries 除 120-tick 数量检测外，在泌乳/乳房 hediff 增删时立即设脏，避免最多 120 tick 的陈旧缓存。
- **涉及**：`CompEquallyMilkable.PoolStorage.cs`（新增 `SetEntriesCacheDirty()`），`MilkRelatedHealthHelper.cs`（EnsureLactatingHediffFromConditions 内 Add/Remove 后调用）。
- **实现**：Comp 暴露 `internal void SetEntriesCacheDirty()`；EnsureLactatingHediffFromConditions 在 GetOrAddHediff / RemoveHediff 后调用 `pawn.CompEquallyMilkable()?.SetEntriesCacheDirty()`。

### B2. flowPerTick / shrinkFactor 入口强制 ≥0、[0,1]
- **目标**：数值稳定性，防止异常 mod/配置导致负流或回缩因子越界。
- **涉及**：`CompEquallyMilkable.Inflow.cs`。
- **实现**：`flowPerTickScale = Mathf.Max(0f, basePerDay / 60000f * 60f)`；`shrinkFactor = Mathf.Clamp01((1f - PoolModelConstants.ShrinkPerStep) * healthPercent)`；单侧 flowPerTick 已用 `Mathf.Max(0f, flowPerTick)`。

### B3. Magic Number 抽到 PoolModelConstants
- **目标**：0.95f（满阈值）、1E-5f（显示判空）、60000f/60f（toPerDay）等抽成常量，便于调参与文档对应。
- **涉及**：`Core/Constants/Constants.cs`（PoolModelConstants），`CompEquallyMilkable.Inflow.cs`，`FluidPoolState.cs`，`MilkRelatedHealthHelper.cs`。
- **实现**：新增常量如 `FullnessThresholdFactor = 0.95f`、`DisplayEpsilon = 1E-5f`、`TicksPerGameDay = 60000f`（已有 Interval60PerDay = 60/60000，可补 TicksPerGameDay）；替换所有字面量。

### B4. TickGrowth 参数封装成 struct BreastPairCapacities
- **目标**：FluidPoolState.TickGrowth 参数过多，封装为 `struct BreastPairCapacities { float LeftBase, RightBase, LeftStretch, RightStretch }`。
- **涉及**：`FluidPoolState.cs`，`CompEquallyMilkable.Inflow.cs`，`FluidPoolStateTests.cs`。
- **实现**：新建 struct（或放在 FluidPoolState 同文件）；TickGrowth(flowLeft, flowRight, capacities)；调用处构建 capacities 传入。

### B5. 存档更健壮：清理时考虑地图上所有 pawn（含尸体）
- **目标**：CleanupDeadPawns 构建有效 loadID 时包含尸体等，防止 loadID 丢失导致 BreastBaseSeverity 被误删。
- **涉及**：`RJWLactatingBreastSize.cs`（CleanupDeadPawns）。
- **实现**：用 `map.mapPawns.AllPawns` 构建 validLoadIds（当前 RimWorld 无 AllPawnsEverAlive，AllPawns 含地图上所有 pawn）。若后续版本提供 AllPawnsEverAlive 可改用。

### B6. Gizmo 显示 P/L/C（调试）
- **目标**：开发模式下选中 pawn 时，Gizmo 显示池 pressure、letdown、conditions、总 overflow/reabsorb，便于调参。
- **涉及**：`CompEquallyMilkable.cs`（重写 CompGetGizmosExtra）。
- **实现**：当 Prefs.DevMode 且 Active 时，添加 Command_Action 或 Info Gizmo，Label/Tooltip 为 "池 P=... L=... C=... overflow=..."。
